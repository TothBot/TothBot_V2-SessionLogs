2.1 DEFECT-SS-004 — Root Cause Confirmed
------------------------------------------------------------

In ws_manager.py, _startup_connect_and_subscribe() paper
mode path (lines 651–671):
  - Sets paper balance
  - Connects public WS
  - Calls _subscribe_public() (which triggers instrument
    subscription, which eventually triggers
    _subscribe_ohlc_channels() via _handle_instrument)
  - Resets zombie timer
  - Returns immediately ← ROOT CAUSE

seed_indicators_from_rest() was only called in TWO places:
  1. Its own definition
  2. Inside _execute_reconnect_sequence() for gap > 15 min

Never called during startup in either live or paper mode.
_subscribe_ohlc_channels() sent subscriptions and logged
OHLC_SUBSCRIBED, then returned. No warm-up launched.
No pair ever reached WARMUP_READY. Pipeline never fired.

------------------------------------------------------------
2.2 OI-NEW-001 — Root Cause Confirmed
------------------------------------------------------------

_subscribe_ohlc_channels() sent TWO separate subscription
messages: interval=5, then interval=60. Kraken WS v2
only allows one ohlc interval per symbol per connection.
interval=60 rejected: "Already subscribed to one ohlc
interval on this symbol." htf_ema_20/htf_ema_50 never
received WS incremental updates after REST seed.
