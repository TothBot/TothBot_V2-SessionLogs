5. WORK DONE (TothBot_V2-Code, commit 0d02278; 3 source + 3 test files, +1407 lines)
   5a. tothbot/exchange/transport.py - the injectable async socket boundary.
       Transport Protocol (async send/recv/close) + WebsocketsTransport adapter
       (JSON encode/decode; maps every library connection-closed / OSError onto a
       single transient TransportClosed). connect() lazily imports the mandatory
       asyncio-native websockets client (WS-LIB-001) with the fixed invariants
       (WS-LIB-002..004 via connection.ws_connect_kwargs); websockets stays a
       VPS-runtime dep, never imported at module load (suite runs without it).
   5b. tothbot/exchange/receive_loop.py - the per-shard steady-state receive loop.
       PURE classify() sorts each frame (PONG/HEARTBEAT/STATUS/SUBSCRIBE_ACK/
       CHANNEL_DATA/UNKNOWN). SYNC handle_message()/on_tick() drive the existing
       pure units: keepalive.mark_real_data (only real data resets zombie; pong
       clears the ping; heartbeat + status do NOT reset zombie), silent-pair
       mark_data/mark_subscribed/evaluate, reconcile.observe_executions/_balances
       (private seq-gap), and DispatchTable routing. rule:HR-WM-012 candle-discard
       gate: an ohlc_5m frame arriving while ANY shard is mid-reconnect is NOT
       routed onward (liveness + silent-pair still applied; only the pipeline fire
       is suppressed). WS-REC-003 reason select: a status maintenance announcement
       before a drop -> Scenario B, else Scenario A. Async run()/_step() is the
       only I/O surface (asyncio.wait_for(recv) with the tick interval as timeout
       so timers fire under a steady stream); a TransportClosed is caught LOCALLY
       and turned into a reconnect for that shard (HR-WM-029), then the loop reads
       from the fresh Transport the reconnect returns. Emits ONLY canonical
       event_registry codes.
   5c. tothbot/exchange/reconnect_driver.py - the async _initiate_reconnect body.
       Wraps the PURE reconnect.py policy. coordinator.begin raises the HR-WM-012
       in-progress gate for the WHOLE reconnect (cleared only on restore success);
       each attempt sleeps reconnect_delay_sec(scenario, attempt) - the CIATS-owned
       Scenario-A backoff seed (5 immediate, 1-16s, 30s cap) or the Scenario-B 5s
       floor - then runs build_restore_sequence(paper_mode): RECONNECT_SOCKET opens
       the fresh socket (returned to the loop), every other step delegated to the
       injected run_step (token / paced re-subscribe / maxratecount reset /
       keepalive resume / silent-pair re-arm / reconcile reset / Position Mirror /
       ticker event_trigger). Paper skips the private-side steps (WM-RECONNECT-019).
       Never abandoned (loss-min): a transient restore failure retries the whole
       sequence after closing the partial socket; ar:AR-080 bounds the rate.
