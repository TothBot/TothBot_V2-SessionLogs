4. WORK DONE (S2c2 - commit 1364d39, 4 files, 607 ins)
   All new units under tothbot/exchange/. Pure/testable, I/O at the edges (the
   async connect + subscribe loop that drives them is the I/O edge, wired later).

     sharding.py - PATH-2 connection sharding (the subscribe fan-out). Pure
       partition: SYMBOLS_PER_CONN_SAFE = 500 (fixed engineering constant,
       WM-SHARD-001); N_conns = ceil(universe_size / 500), integer-only ceil,
       floored at 1 so the global channels always have a home (AR-038 startup);
       shard_index = i % N_conns (alternating-index, balanced +/-1). GLOBAL_
       CHANNELS (instrument, status) subscribe ONCE on shard 0 only; PER_PAIR_
       CHANNELS (ohlc_5m, ticker) follow each pair to shard i%N; shard 0 is the
       system-clock shard. ShardAssignment (per-shard pairs + channels +
       subscribe_count) and ShardPlan (builds N shards over the ordered universe
       + the pair_to_shard_index map). rule:HR-WM-029 shard independence (each
       shard owns a disjoint pair set); rule:HR-WM-031 / anti-ar:AR-070 every
       universe pair assigned exactly once, never dropped (subscribe-count
       decoupled from connection count).

     pacing.py - the process-singleton subscribe token bucket (contract:WM-PACE-
       001/002). SubscribeTokenBucket is a pure unit over an injected monotonic
       clock (mirrors keepalive.py): SUBSCRIBE_RATE_PER_SEC = 10, SUBSCRIBE_
       BURST_CAPACITY = 20; starts full (the allowed startup burst), refills 10
       tok/sec capped at capacity; try_acquire() consumes one token if available,
       time_until_next() returns the seconds the I/O edge sleeps before retrying,
       available() exposes the live level for the WM-PACE-005 state log. ONE
       bucket governs EVERY shard's subscribes (the ar:AR-080 Cloudflare
       150-subscribes/10-min ceiling is per-IP for the whole process) and is
       deliberately NOT reset on a shard reconnect, so paced re-subscribes draw
       from the same bucket; no bypass, no priority lane. A backward clock step
       accrues nothing and never drains the bucket.

     tests/test_sharding.py - 21 tests: SYMBOLS_PER_CONN_SAFE=500; global +
       per-pair channel sets; N_conns ceiling table (0/1/499/500->1, 501/700/
       1000->2, 1001->3) + floor-at-1 + negative-reject; shard_index = i%N +
       arg guards; N=2 partition by index parity; shard sizes balanced within 1
       (701 -> 351/350); shard 0 carries globals only; every shard carries the
       per-pair channels; shard 0 is the clock shard; pair_to_shard_index round
       trip; HR-WM-031 every pair assigned exactly once + subscribe-count
       decoupling; single-shard (<=500) carries everything; empty universe still
       has the clock shard with globals.

     tests/test_pacing.py - 17 tests: constants match the diagram + WM-PACE-005
       key; starts full at burst capacity; burst of 20 then empty; refill 10/sec;
       refill caps at capacity; fractional partial refill; time_until_next = 0
       when a token is available, 0.1s when empty, 0.05s after a 0.5-token
       partial; acquire succeeds after waiting the indicated time; backward clock
       step does not drain; construction rejects non-positive rate/capacity.
