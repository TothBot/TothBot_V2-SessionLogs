3. DECISION RATIFIED THIS SESSION - OPTION A
   (ARCHITECTURE vs TESTING SEPARATION)
------------------------------------------------------------

CONTEXT.
The TB00396 NSI (already committed at TB00395 close)
scoped Phase 1 as: "migrate parameter VALUES out of
TB00000 sec 9.17 into the 0500000 diagram notes, then
RETIRE the sec 9.17 entry." Bill challenged this on the
established principle that testing must remain separate
from architecture.

FP/DP ANALYSIS (performed this session).
  First principle:
    - Architecture = system structure and contracts
      (components, connections, gates, flows). Permanent;
      changes only on design decisions.
    - Parameter starting values = transient testing seeds.
      Per TB00000 sec 9.16 + sec 6.8 every starting value
      except the 1:1.5 R:R floor is explicitly temporary;
      CIATS owns them; paper trading replaces them.
  Decision principle (minimize loss / minimize rework):
    Migrating transient starting values into the canonical
    0500000 diagrams would (a) pollute the canonical
    architecture source with non-canonical, frequently-
    changing content; (b) make 0500000 (highest-cost,
    highest-risk document to edit) stale the instant CIATS
    changes a value - reintroducing the prose-driven-error
    class the campaign exists to eliminate, relocated into
    the diagrams; (c) violate the already-established
    testing-vs-architecture separation. The "complete and
    comprehensive" mandate is about ARCHITECTURAL
    completeness, not absorption of transient operational
    state.

DECISION (Option A) - RATIFIED BY BILL THIS SESSION.
  - Only ARCHITECTURAL content migrates into 0500000:
    component existence, ownership, contracts, information
    flows.
  - Parameter STARTING VALUES stay in TB00000 sec 9.17.
    sec 9.17 is the canonical, permanent parameter
    registry and is NOT retired.
  - 0500000 records the architectural FACT that a
    parameter exists and who owns it (CIATS), not the
    transient numeric value.
  - The TB00396 NSI Phase 1 scope ("migrate values then
    retire sec 9.17") is SUPERSEDED and must be corrected
    in the campaign plan.

This decision is to be recorded as a Decision Register
entry (DEC-039) and encoded in the campaign plan amendment
NEXT session. NOT recorded this session (deferral per
Bill directive).
