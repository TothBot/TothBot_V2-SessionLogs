2.6 OI-041 CODE-SIDE ROOT CAUSE LOCKED -
    DEFECT-WM-REST-SIGN-001
------------------------------------------------------------

Per Bill's late-session redirect, Claude read
ws_manager.py and cross-checked against 0411002
Kraken_REST_API_Specification dv1_1. The defect was
located in under one minute of code reading.

LOCATED DEFECT:

File:     ws_manager.py (project folder dv1_21; VPS
          running dv1_22 - both have the defect as
          dv1_22 only added paper-mode skip branches,
          not signing)
Sites:    Three private REST call sites:

  Site 1 - GetWebSocketsToken (lines 602-629):

    async def _rest_get_ws_token(self) -> str:
        trade_key = self._config["kraken_trade_api_key"]
        trade_secret = self._config["kraken_trade_api_secret"]
        async with self._make_http_session() as session:
            resp = await session.post(
                f"{REST_BASE_URL}/0/private/GetWebSocketsToken",
                headers={"API-Key": trade_key},
                data={"nonce": str(int(time.time() * 1000))},
            )
            ...

  Site 2 - OpenOrders (lines 656-666):

    async def _rest_get_open_orders(self) -> dict:
        trade_key = self._config["kraken_trade_api_key"]
        async with self._make_http_session() as session:
            resp = await session.post(
                f"{REST_BASE_URL}/0/private/OpenOrders",
                headers={"API-Key": trade_key},
                data={"nonce": str(int(time.time() * 1000))},
            )
            ...

  Site 3 - Balance (lines 668-678):

    async def _rest_get_account_balance(self) -> dict:
        trade_key = self._config["kraken_trade_api_key"]
        async with self._make_http_session() as session:
            resp = await session.post(
                f"{REST_BASE_URL}/0/private/Balance",
                headers={"API-Key": trade_key},
                data={"nonce": str(int(time.time() * 1000))},
            )
            ...

DEFECT NATURE:

All three private REST calls ship with ONLY the
API-Key header and the nonce. They do NOT compute or
attach the mandatory API-Sign HMAC-SHA512 signature.
Site 1 retrieves `trade_secret` (line 609) and then
orphans it - no code path uses it. Sites 2 and 3 do
not even retrieve the secret.

Kraken correctly rejects every such request with
`EAPI:Invalid key` because an unsigned private call
cannot be authenticated. The error name is slightly
misleading: the key itself is valid, but the REQUEST
is unsigned, and Kraken returns `EAPI:Invalid key`
as the standard error code for all auth failures on
private endpoints.

SPEC VS CODE CONTRADICTION:

0411002 Kraken_REST_API_Specification dv1_1 mandates
signing for every private call:

  REST-AUTH-001  Kraken Spot REST private calls use
                 HMAC-SHA512 authentication.
                 Signature formula:
                   message = URI_path +
                             SHA256(nonce + POST_data_string)
                   signature = HMAC-SHA512(message,
                               base64_decode(api_secret))
                   API-Sign header = base64_encode(signature)

  REST-AUTH-003  Required headers for all private REST calls:
                   API-Key:  TRADE API key value
                   API-Sign: HMAC-SHA512 signature (above)
                 Content-Type: application/x-www-form-urlencoded

The code at all three sites is in direct violation of
REST-AUTH-001 and REST-AUTH-003. 1011002 WS_Manager_
Coding_Spec contains rules WM-TOKEN-001 / WM-TOKEN-002
referenced in the _rest_get_ws_token docstring but
these rules did not explicitly enumerate signing
mechanics (per-spec review needed in TB00121 DC
package).

OI-041 ROOT CAUSE:

DEFECT-WM-REST-SIGN-001. Three unsigned private REST
calls in ws_manager.py. Fix scope: add HMAC-SHA512
signing helper, apply to all three call sites, update
1011002 WS_Manager_Coding_Spec with explicit
WM-REST-SIGN-00X rules, bump 1010000 registry. Full DC
package required per NO PATCHES rule. See TB00120 NSI
v1_1 §6 for the fix file list.

OPERATIONAL IMPACT:

Today (paper mode): ZERO. dv1_22 added paper-mode
early-return branches that skip all three unsigned
REST calls before they dispatch. Service runs without
error.

At live-mode transition (PAPER_TRADING_MODE=false):
100% failure. Every entry, every balance check, every
order query would fail with EAPI:Invalid key. Live
trading is blocked until fix ships.
