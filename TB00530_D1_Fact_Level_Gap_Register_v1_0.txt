TB00530 D1 FACT-LEVEL GAP REGISTER
0411001 dv1_13  vs  0500000 dv1_142 D1
============================================================

PURPOSE
  The exhaustive fact-level gap register for D1 (Kraken API
  Connection Architecture, figure R9 in 0500000 dv1_142). It
  enumerates every architectural wire-fact carried by 0411001
  dv1_13 across all five SD-9-verified contract groups and
  marks each PRESENT-IN-D1, ABSENT, or CONFLICT, where
  "in D1" means derivable from the D1 figure ALONE per the
  strict DD-014 standard (SVG visual + governs <desc>
  descriptors + figcaption ONLY; NO 0500000 prose, NO
  section-8 AR registry, NO subordinate spec).

  This register is the corrected fact-level replacement for
  the over-claimed D1 V-040 65-cell "design-complete"
  attestation (TB00376; flagged over-claimed in TB00529).
  It is the input that drives the D1 diagram-first migration
  sessions (CF-D1-MIGRATE) and the eventual 0411001
  retirement. Read-only: no 0500000 edit, no retirement.

METHOD
  Source of truth = GitHub HEAD via public tarball (DEC-097).
  The D1 "diagram-alone" corpus = the 14 <desc> governs blocks
  (1 root + 13 components) + the figcaption + the visible SVG
  <text> labels of the D1 figure, decoded from the live
  0500000 dv1_142 HTML. Each 0411001 dv1_13 WS-* wire-fact was
  checked for derivability against that corpus ONLY.

  NOTE ON VERSION: the inbound NSI/figcaption cite D1 at
  dv1_141 / dv1_83 in places; the live HEAD figure is dv1_142
  (figure title R9). This register is built against live HEAD
  dv1_142 per the TB00000 section 2 reading rule. The dv1_141
  cl_ord_id/req_id desc addition IS present in the dv1_142
  D1 desc and is counted.

3. GAP REGISTER - CONFLICT SET (2)
============================================================
  A CONFLICT is a wire-fact whose token IS present in the D1
  visible text but whose VALUE contradicts the verified
  0411001 dv1_13 fact. A coder reading D1 alone would code
  the WRONG value. Conflicts are higher-severity than plain
  absences and must be resolved diagram-side (DD-002).

  GROUP 5 - KEEPALIVE / ZOMBIE / RECONNECT / RATE LIMIT
    WS-REC-002    WS_Manager                 reconnect scenario A random: 5 immediate then backoff
                  CONFLICT: D1 visible text says '10 attempts, backoff 1s->30s'; spec says 5 immediate then backoff -> value conflict
    WS-RL-002     WS_Manager                 decay 2.34/sec, max 80 open/pair  [LOSS-PREVENTION]
                  CONFLICT: D1 visible hardcodes 'Threshold per pair:125'; spec WS-RL-003 mandates maxratecount, do NOT hardcode 125 (AR-030)
