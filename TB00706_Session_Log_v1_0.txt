   All new units under tothbot/exchange/. Pure/testable, I/O at the edges (the
   async read loop that drives them is the I/O edge, wired in a later session).

     keepalive.py - mod:WS_Manager connection-liveness (rule:HR-WM-003 ping +
       rule:HR-WM-004 zombie). ConnectionKeepalive is a pure timer over an
       injected monotonic clock. PING/PONG (A-7 / WS-PING-002): send the
       application JSON ping {"method":"ping"} every 30s (PING_INTERVAL_SEC);
       a ping with no pong within 10s (PONG_TIMEOUT_SEC) = DEAD connection ->
       reconnect; a ping is never stacked on an unanswered one. ZOMBIE (A-8 /
       WS-ZOM-003): last_real_data_time tracked via time.monotonic; ONLY real
       market-data events reset it (pong AND heartbeat do NOT); elapsed > 90s
       (ZOMBIE_THRESHOLD_SEC) = ZOMBIE -> log ZOMBIE_CONNECTION_DETECTED,
       reconnect. liveness() returns ALIVE / DEAD_NO_PONG / ZOMBIE (the acuter
       10s pong timeout reported ahead of the 90s zombie). reset() re-arms all
       timers on every (re)connect (WS-REC-004 resumes the ping + zombie tasks).

     reconcile.py - contract:Reconciliation_REST sequence-gap detection (A-9 /
       A-10). SequenceGapDetector tracks the last-seen sequence for ONE channel;
       a forward gap (received_seq > last_seq + 1) returns a SequenceGap naming
       the recovery to run; the first message is the baseline; duplicate /
       out-of-order (seq <= last_seq) is not a forward gap and never moves the
       watermark backwards; after a gap the watermark advances so there is no
       re-alert on the following in-order messages. Per-channel wiring:
       executions -> alert EXECUTIONS_SEQUENCE_GAP, REST GetOpenOrders, severity
       HIGH (WS-EXE-019; a missed fill leaves a live position unmanaged);
       balances -> alert BALANCES_SEQUENCE_GAP, REST GetAccountBalance, severity
       MEDIUM (WS-BAL-005). ReconciliationTracker owns the two INDEPENDENT
       detectors; reset() clears both on reconnect (A-10: sequence tracking
       resets on each new subscription; no cross-session gap-detection). The
       REST recovery call itself is the I/O edge, driven later; this module owns
       detection and names the endpoint.

     tests/test_keepalive.py - 11 tests: constants match the diagram; 30s ping
       cadence; ping-not-stacked-while-awaiting-pong; pong clears + reschedules;
       no-pong-in-10s = DEAD; pong-before-deadline stays ALIVE; 90s zombie
       threshold; real-data resets the zombie timer; pong does NOT reset zombie;
       dead-pong reported ahead of zombie; reset clears outstanding ping + zombie.

     tests/test_reconcile.py - 10 tests: first-message baseline; clean advance;
       forward-gap detected with missed count; no re-alert after gap advance;
       duplicate/out-of-order is not a gap; executions -> GetOpenOrders/HIGH;
       balances -> GetAccountBalance/MEDIUM; reset drops the watermark; tracker
       channels independent; tracker reset clears both.
