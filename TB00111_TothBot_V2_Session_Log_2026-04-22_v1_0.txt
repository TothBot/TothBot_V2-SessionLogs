  PowerShell's backslash-escaping of embedded double
  quotes for a single-line ssh command does not survive

2.5 Sweep script defect — top-line FATAL count
------------------------------------------------------------

The top-line "FATAL events: 0" reading was false
reassurance. The 4 new WS_MGR_FATAL events are logged
at level=CRITICAL, not level=FATAL. The sweep script's
tripwire-#1 grep pattern '"level":"FATAL"' does not
match level=CRITICAL events and therefore missed the
regression. The bottom-section WS_MGR_FATAL counter
caught it (6 -> 10 delta).

TB00110 NSI §H-2 tripwire #1 reads "Any FATAL event".
The NSI author (prior Claude Chat) likely meant the
word generically. The code actually emits reconnect
failures at level=CRITICAL. This is a semantic
disconnect between the NSI wording and the code
behavior — to be resolved in TB00112.

Sweep script remediation options (for TB00112):
  Option A: Broaden pattern to match FATAL or CRITICAL.
            Risk — CRITICAL level is used for other
            events too, increasing false-positive rate.
  Option B: Match event names directly (WS_MGR_FATAL,
            other enumerated FATAL-class events).
            More precise but requires enumerating all
            FATAL-class event names.
  Option C: Both — CRITICAL|FATAL level match AND
            cross-check against WS_MGR_FATAL count.

Recommendation: Option B is preferred per first-
principles precision. Decision deferred to TB00112.

------------------------------------------------------------
6.1 The PS -> ssh -> bash quoting problem is SOLVED
------------------------------------------------------------

Solution: put bash grep patterns in a script on the VPS.
PowerShell passes only the script path. No escape transit.
Record of proven approach:

  scp <local> root@<ip>:/root/<script>
  ssh root@<ip> "chmod +x /root/<script>"
  ssh root@<ip> "sed -i 's/\r$//' /root/<script>"  (CRLF fix)
  ssh root@<ip> /root/<script>                     (execute)

Adopted as standard pattern for all future monitoring
that would otherwise need nested quoting.

------------------------------------------------------------
6.2 Sweep-script counters must be derived from code truth
------------------------------------------------------------

Observation: monitoring patterns written from memory of
NSI wording can disagree with what the code actually
emits. The top-line FATAL counter matched the NSI
wording but missed the real CRITICAL-level events.

Lesson: pattern definitions for monitoring scripts must
be derived by reading the logger.py / ws_manager.py
event-emission code, NOT by paraphrasing the NSI.
Record in TB00112 as an operational engineering rule.
