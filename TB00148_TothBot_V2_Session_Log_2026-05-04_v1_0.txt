5. PATTERN ANALYSIS
============================================================

Defect concentration is NOT random. The 24 findings cluster
tightly at four architectural seams:

  CLUSTER 1 — PAPER-MODE DISPATCH BOUNDARY (PA-004 violation)
    Findings: 09, 29, 30
    EE + LM unconditionally send to ws_private. paper_trading_mode
    flag not threaded. PA-004 boundary unimplemented at code layer.
    Root cause: spec gap. 1011004 (Execution Engine spec) MISSING.
    No spec defined the dispatch contract for the paper case.

  CLUSTER 2 — POSITION MIRROR INTEGRATION (HR-PM-009 violation)
    Findings: 10, 14, 15, 16, 21
    Four conflicting PositionMirror access patterns across
    EE/EC/LM. None match the 1011006 dv1_1 HR-PM-009
    "ws_manager is sole writer" contract. PositionRecord
    schema missing required fields (fees_entry_usd,
    mae_pct_reached). Layer 2 MAE-threshold exits non-functional.

  CLUSTER 3 — CIATS PARAMETER WIRING
    Findings: 12, 18, 24, 25, 26
    Multiple CIATS-owned parameters hardcoded as module
    constants. Half-Kelly persistence missing. Regime engine
    has 3 hardcoded thresholds. Net effect: CIATS proposals
    will compute correctly, log correctly, but never take
    effect. Continuous improvement loop is broken.

  CLUSTER 4 — CROSS-MODULE INTERFACE DRIFT
    Findings: 19, 22, 23, 27, 28, 32
    pair_specs vs pair_cache attribute drift. Encapsulation
    violations (cross-module private attr reads). Trade
    Outcome Bus field naming drift. Wire-up ordering
    assumptions not documented.

The four clusters share a common origin: rapid spec/code
co-evolution without architecture-first discipline. Each
cluster is an integration boundary, not a calculation
defect. Indicators, gates, regime taxonomy, and statistical
methods are all internally correct. The problem is at the
seams between modules, not within them.

This pattern strongly favors OPTION B (clean-slate rebuild
with reuse) over OPTION A (surgical fix): clusters require
coordinated re-design at the seam, not 24 independent patches.
