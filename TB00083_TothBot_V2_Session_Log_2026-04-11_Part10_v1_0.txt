------------------------------------------------------------
3.2 CommitHash Circular Defect — Identified and Corrected
------------------------------------------------------------

After the first GitHub commit (52653e5), Bill identified
a design defect: requiring a git commit hash in 1010000
after every TothBot_V2-Code commit forced a second commit
to TothBot_V2-Docs to update 1010000 — which itself
requires no further registry update but creates a never-
ending loop pattern whenever any Code commit is made.

Root cause: CommitHash in the Code File Registry is
redundant with git. Git provides commit history natively
via git log. The field added maintenance burden without
adding traceability benefit.

Fix: CommitHash removed from 1010000 Section 5 registry
format and all registry entries. Section 5.6 rewritten
as Registry Update Procedure — update only on file
additions, removals, or spec version changes, not after
every commit.

0311002 updated to v1_2 with STEP 8.2.1 corrected.
1010000 updated to dv1_3 with CommitHash fields removed.
