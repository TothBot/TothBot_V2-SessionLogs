2.3 Root Cause Identified — DEFECT-SS-007
------------------------------------------------------------

DEFECT ID:   DEFECT-SS-007
TITLE:       _wsname_to_classic empty at startup —
             _rest_get_ticker() returns {} immediately
ROOT CAUSE:
  WM-LIQ-003 (1011002 dv1_15) states:
    "Build TWO maps at startup from instrument snapshot
    (which provides both formats)"
  THIS IS WRONG. Kraken WS v2 instrument snapshot does
  NOT include an altname field. The instrument channel
  snapshot returns only: symbol, base, quote, status,
  qty_precision, qty_increment, price_increment,
  cost_min, qty_min, marginable, has_index,
  cost_precision. No altname. No classic REST name.

  Therefore: pair.get("altname", "") always returns "".
  _wsname_to_classic stays empty.

  In _rest_get_ticker():
    altname_to_wsname dict stays empty.
    "if not altname_to_wsname: return {}" fires.
    No REST Ticker call is made.
    No LIQUIDITY_REFRESH_FAILED is logged (silent exit).
    Returns {}.

  In _warm_up_all_pairs():
    Both tries of _rest_get_ticker() return {}.
    Retry after 2s also returns {}.
    LIQUIDITY_SEED_FAILED logged.
    liquidity_24h stays all zeros.
    Gate 2 blocks all entries every candle forever.

CORRECT DATA SOURCE:
  REST /0/public/AssetPairs endpoint.
  Returns both wsname ("XBT/USD") and altname ("XBTUSD").
  NOTE: AssetPairs wsname uses "XBT/USD".
        WS v2 uses "BTC/USD".
        Normalization required: XBT/ -> BTC/, /XBT -> /BTC.

FIX DESIGN:
  Add _rest_build_pair_maps() method:
    Calls GET /0/public/AssetPairs (public, no auth).
    For each pair in result:
      altname = info.get("altname", "")
      wsname_raw = info.get("wsname", "")
      Normalize: ws_v2_name = wsname_raw.replace
        ("XBT/","BTC/").replace("/XBT","/BTC")
      If ws_v2_name in self.pair_cache:
        self._wsname_to_classic[ws_v2_name] = altname
        self._classic_to_wsname[altname] = ws_v2_name
        norm = self._normalize_classic_key(classic_key)
        self._classic_to_wsname[norm] = ws_v2_name
    Logs PAIR_MAPS_BUILT(pairs_mapped=N).
    On any exception: logs PAIR_MAP_BUILD_FAILED, returns.
    Gate 2 blocks if maps stay empty (fail-safe preserved).

  Modify _warm_up_all_pairs():
    Call await self._rest_build_pair_maps() BEFORE
    the existing call to self._rest_get_ticker(candidates).

  Update 1011002 WS_Manager_Coding_Spec:
    WM-LIQ-003: Correct the data source.
    Remove "instrument snapshot provides both formats."
    Replace with REST AssetPairs as authoritative source.
    Add WM-LIQ-011: _rest_build_pair_maps() specification.
    Add XBT/BTC normalization rule.
