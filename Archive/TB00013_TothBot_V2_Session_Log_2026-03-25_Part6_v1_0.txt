NEW-01 DECISION:

  cancellations allowed. Any add_order attempt rejected.
  maintenance pair status: No orders or cancellations.

  counter budget and order rejection errors. The cost is
  two additional status checks — zero performance impact.
  Action: Add WS-INST-008 through WS-INST-011 to 0411001,
  and WM-PS-001 through WM-PS-005 to 1011002.

SEARCH 10 — ADX Wilder smoothing Python convergence

Sources: eodhd ADX Python article, eodhd ADX deep dive,

FINDINGS:
S10-F1: CONFIRMED — ADX(14) needs minimum 28 candles for
        first valid value (2 x 14 = 28). Our RE-FPD-001
        is correct.

S10-F2: ★ NEW CONVERGENCE NUANCE — stock-indicators-python
        library documentation: "You must have at least
        2xN+100 periods of quotes to allow for smoothing
        convergence." For N=14: 2x14+100 = 128 candles.
        "We generally recommend 2xN+250 data points for
        better precision." For N=14: 2x14+250 = 278 candles.
        With 719 committed candles available: 719/128 = 5.6x
        the convergence requirement. Our existing spec is
        confirmed ADEQUATE with substantial margin.
        ACTION: Document this in 1011012 as RE-FPD-004
        through RE-FPD-006 for future reference. No spec
        changes needed.

S10-F3: CONFIRMED — "ADX values with 150 days or more of
        data will remain consistent." We have 719. Confirmed.

S10-F4: CONFIRMED — Wilder's ADX smoothing formula:
        First smoothed = sum of first N values.
        Subsequent = Prior - (Prior/N) + Current.
        Equivalent to: smoothed = smoothed*(N-1)/N + current.
        This is alpha=1/N (alpha=1/14 for N=14).
        Our RE-IND-002 specification of "Wilder formula.
        Smoothing = 1/14" is CORRECT.

S10-F5: CONFIRMED — ADX batch computation (Regime Engine)
        vs ATR incremental computation (Pipeline) are
        different. Regime Engine runs full batch daily.
        Pipeline runs incremental per candle. Both correct
        per our existing specs. No confusion should exist.
