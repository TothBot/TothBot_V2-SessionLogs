4. NEW ARCHITECTURAL DECISIONS
============================================================

AR-054 — PARTIAL FILL + EXPIRY/CANCEL POSITION HANDLING:
  Execution dispatch table MUST handle these cases for
  entry orders (identified by cl_ord_id in pending_orders):

  CASE A: exec_type == "expired" AND cum_qty > 0:
    This is a partially-filled entry order that expired.
    A real position exists. Must protect it immediately.
    1. Remove cl_ord_id from Pending Order Registry (AR-053)
    2. Compute actual position parameters:
       qty = Decimal(str(event.cum_qty))
       entry_price = Decimal(str(event.avg_price))
    3. Recalculate TP and emergSL prices using actual
       entry_price (not original limit_price):
       mae_dist = ATR(14) × mae_mult
       emergSL_dist = ATR(14) × emergency_sl_mult
       net_loss = mae_dist / entry_price + 0.0016 + 0.0026
       net_gain = net_loss × 1.5
       gross_tp = entry_price × (1 + net_gain + 0.0016)
       tp_price = gross_tp.quantize(price_incr, ROUND_UP)
       emergSL_trigger = (entry_price - emergSL_dist)
                         .quantize(price_incr, ROUND_DOWN)
    4. Validate R:R still net 1:1.5 (it will be, since
       ATR values haven't changed)
    5. Verify qty >= qty_min and qty × entry_price >= cost_min
       If not: do NOT dispatch TP/emergSL. Instead:
         Issue market sell for cum_qty immediately.
         Log PARTIAL_FILL_BELOW_MINIMUM.
         Fire CIATS Trade Outcome Bus with trade data.
    6. dispatch batch_add:
       Order 1: take-profit limit, sell, qty, tp_price
       Order 2: stop-loss (emergSL), sell, qty,
                trigger=emergSL_trigger, reference=last
    7. Add position to Position Mirror with cum_qty, entry_price
    8. Log ENTRY_PARTIAL_FILL_PROTECTED.

  CASE B: exec_type == "canceled" AND cum_qty > 0:
    Same as Case A. Possible reason: post_only rejected
    after partial fill (edge case). Same treatment.

  CASE C: exec_type == "expired" OR "canceled" AND cum_qty == 0:
    Entry order expired/canceled with no fill.
    Remove from Pending Order Registry. Log. No position.

  NOTE: exec_type == "filled" (complete fill) handling
  is unchanged. No partial fill concern for "filled".

  Assign: 1011002 WM-DISP-020 through -028.

AR-055 — DEAD MAN'S SWITCH: EXPLICITLY PROHIBITED:
  TothBot V2 MUST NOT call cancel_all_orders_after at
  any time during normal operation or shutdown.
  RATIONALE:
    DMS cancels ALL orders in account when timer expires.
    This includes emergSL orders protecting open positions
    (Layer 3 crash protection). Canceling emergSL leaves
    open positions completely unprotected during VPS/
    TothBot/internet outage — exactly the scenario
    emergSL is designed to protect against.
    DMS removes the protection it is meant to provide.
  PROTECTION ACHIEVED WITHOUT DMS:
    Entry order protection: GTD self-expiry in 30-60s.
    Open position protection: emergSL resting on Kraken.
    Process crash protection: systemd WatchdogSec=120.
    Network breakdown: emergSL remains on Kraken exchange.
  HARD RULE: No call to cancel_all_orders_after is
  permitted in TothBot V2 code under any circumstance.
  Code review must reject any PR containing this call.
  Assign: HR-DMS-001 in 1011002 and 0411001.

AR-056 — MID-SESSION WS RECONNECT SEQUENCE:
  Add Section 2 to 1011014 Startup_Sequence_Coding_Spec.
  This is distinct from startup (Section 1).

  Trigger: private WS connection drops during live
  operation. WS Manager detects via asyncio exception
  or timeout.

  Reconnect steps:
  Step R1: Close broken WS connection cleanly.
  Step R2: Wait — if first few retries, instant;
           if extended outage, 5s interval.
  Step R3: Re-request GetWebSocketsToken (token may
           have expired during gap).
  Step R4: Re-connect private WS.
  Step R5: Re-subscribe executions (snap_orders:true,
           snap_trades:true, order_status:true,
           ratecounter:true).
  Step R6: Process snap_orders:
    a) Compare snap_orders open orders vs Position Mirror.
    b) For each cl_ord_id in Position Mirror but NOT
       in snap_orders (position closed during gap):
       Find corresponding trade in snap_trades by order_id.
       Fire CIATS Trade Outcome Bus. Update Position Mirror.
       Log POSITION_CLOSED_DURING_GAP.
    c) For each order in snap_orders but NOT in Position
       Mirror (order placed by error/manual): alert only.
    d) For orders in Pending Order Registry: if now in
       snap_orders as open → remove from registry (placed
       before disconnect, now confirmed live).
       If NOT in snap_orders → clear from registry
       (expired or never placed). Log PENDING_ORDER_LOST.
  Step R7: Re-subscribe balances channel (snapshot:true).
    Update spot_asset_balances from new snapshot.
    DO NOT update portfolio_baseline_USD.
  Step R8: Re-subscribe to public channels:
    instrument, status, ohlc(5m), ohlc(60m).
    Update pair cache from new instrument snapshot.
  Step R9: Re-subscribe ticker for all open positions
    (event_trigger:bbo for each active pair).
  Step R10: If gap > 15 minutes:
    Re-run OHLC REST warm-up for stale pairs only.
    Check if ATR/EMA caches are stale (> 15 min since
    last update). If stale → re-warm affected pairs.
    If gap <= 15 minutes: skip REST warm-up.
  Step R11: Resume pipeline. Set all WARM_UP pairs to
    READY (if not stale). Log RECONNECT_COMPLETE.

  KEY INVARIANTS (must be preserved across reconnect):
  - portfolio_baseline_USD: NEVER reset during reconnect.
  - drawdown state: PRESERVED.
  - ATR/EMA caches: PRESERVED unless stale.
  - Position Mirror: updated by reconciliation only.

  Assign: 1011014 Section 2. 1011002 WM-RECONNECT-001
  through -015.
