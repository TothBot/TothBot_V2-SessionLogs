SEARCH 4 — Python Decimal vs float for price rounding
------------------------------------------------------------

Sources: Python decimal docs (official), Python rounding
tutorial (Real Python), GeeksforGeeks quantize(), Decimal
finance guide (LabEx), Decimal finance 2026 best practices,
Decimal quantize bomberbot, coderivers, Python precision
finance, mojoauth Decimal guide, purpletutor Python decimal,
note.nkmk.me floor/ceil, TechAlmirah floor/ceil, Python
Engineer precision, upGrad floor functions.
Unique references: ~15 new

FINDINGS:
S4-F1: ★ NEW-12 CRITICAL — AR-047 (prior session) specifies
       `math.floor(raw_qty / qty_increment) × qty_increment`
       for rounding. math.floor() operates on Python floats.
       Float binary representation causes errors:
       `math.floor(42350.15 / 0.1) * 0.1 = 42350.1000...0` 
       appears correct but `math.floor(0.3 / 0.1) * 0.1`
       may give 0.2 instead of 0.3 due to float imprecision.
       For exchange order validation, wrong rounding = 
       rejected order = failed trade.

S4-F2: ★ NEW-12 — Correct pattern:
       `Decimal(str(raw_price)).quantize(
           Decimal(str(price_increment)),
           rounding=ROUND_FLOOR
       )`
       `str(value)` conversion preserves exact decimal
       representation. NEVER `Decimal(float_value)`.

S4-F3: CONFIRMED — ROUND_FLOOR = always toward negative
       infinity. For positive prices: same as truncation.
       ROUND_CEILING = always toward positive infinity.
       Correct for our use case.
       
S4-F4: CONFIRMED — All Kraken API values (prices/qtys) come
       as Python number (float) from JSON. Must convert to
       `Decimal(str(value))` immediately on WS/REST receipt.
       BP-JSON-005 handles Decimal serialization. Needs
       complementary BP-DEC-001 for deserialization/conversion.

SEARCH 4 CONCLUSION: NEW-12 confirmed. AR-047 correction
needed. All price/qty rounding must use Decimal.quantize().

AR-050 — BALANCES CHANNEL: USE SPOT WALLET ONLY:
  TothBot maintains `spot_asset_balances: dict[str, Decimal]`
  Updated from balances channel events:
  SNAPSHOT: for each item in data, find wallet where
    type=="spot" AND id=="main". Extract balance.
    spot_asset_balances[asset] = Decimal(str(wallet.balance))
  UPDATE: if wallet_type=="spot" AND wallet_id=="main":
    spot_asset_balances[asset] = Decimal(str(event.balance))
    # event.balance = balance AFTER this transaction
  NEVER use top-level `balance` field in snapshot.
  The top-level balance includes "earn" and other wallets
  which are NOT available for trading.
  spot_usd_balance = spot_asset_balances.get("USD", Decimal("0"))
  Assign: 1011002 WM-BAL-001 through -006.

AR-051 — PORTFOLIO VALUE AND POSITION SIZING DEFINITION:
  For spot long-only trading with USD/USDT/USDC quote pairs:
  portfolio_USD = spot_usd_balance (AR-050, cash in hand)
  tradeable_USD = portfolio_USD × tradeable_pct (CIATS-owned)
  per_trade_USD = tradeable_USD × per_trade_pct (CIATS-owned)
  Position size: raw_qty = per_trade_USD / entry_limit_price
  Apply AR-047 Decimal rounding to raw_qty before dispatch.
  Half-Kelly at 200+ trades (from CIATS):
    kelly_pct = W - (1-W)/R, apply 50%
    per_trade_USD = min(kelly_pct × tradeable_USD,
                       tradeable_USD / max_concurrent)
  Portfolio value (for CIATS trade outcome bus):
    realized_portfolio = spot_usd_balance
  Portfolio value (for drawdown monitoring, AR-052):
    includes open position mark-to-market.
  Assign: 1011002 WM-PS-005 through -009,
          1011011 RE-SZ-001 through -005.

AR-052 — PORTFOLIO DRAWDOWN FORMULA:
  portfolio_baseline_USD: set in Startup Step 6 (after
  snap_orders reconciliation), equal to spot_usd_balance
  at that moment. Never updated during operation.
  Resets only on TothBot restart.

  current_portfolio_USD = spot_usd_balance
    + sum(bid[symbol] × qty for all open longs in Position Mirror)
  Uses bid price from ticker (per AR-048).

  drawdown_pct = max(0,
    (portfolio_baseline_USD - current_portfolio_USD)
    / portfolio_baseline_USD)

  SESSION PAUSE (Gate 7, ticker-driven continuous check):
    If drawdown_pct >= session_pause_drawdown:
      gate_state = SESSION_PAUSED
      Block all new ENTRY orders.
      Do NOT close existing open positions.
      Log SESSION_PAUSE_DRAWDOWN.
      Alert operator.
      Resume when drawdown_pct < session_pause_drawdown
      AND operator has been alerted.

  FULL HALT (Gate 7):
    If drawdown_pct >= full_halt_drawdown:
      gate_state = FULL_HALT
      Block all new ENTRY orders.
      Do NOT close existing positions.
      Log FULL_HALT_DRAWDOWN.
      Alert operator.
      DOES NOT auto-resume. Requires operator restart.

  Rationale: Forced sells at halt time would violate R:R
  and potentially worsen the drawdown. Resting emergSL
  provides crash protection for existing positions.
  CIATS will study halt patterns and add PDCA improvements.
  Assign: 1011002 WM-DD-001 through -008,
          1011011 RE-DD-001 through -006.

AR-053 — PENDING ORDER REGISTRY:
  WS Manager maintains:
    pending_orders: dict[str, Decimal]  # cl_ord_id → USD cost
  Populated: when entry add_order is DISPATCHED:
    estimated_cost = order_qty × entry_limit_price
                   + order_qty × entry_limit_price × 0.0016
                   # qty × price × (1 + maker_fee)
    pending_orders[cl_ord_id] = estimated_cost
  Cleared: when executions channel receives event for
    that cl_ord_id with exec_type in:
    {filled, expired, canceled, rejected}

  Gate 7 check addition:
    available_USD = spot_usd_balance - sum(pending_orders.values())
    if available_USD < per_trade_USD:
      SKIP → INSUFFICIENT_USD
      Log with reason code.

  This prevents overcommitment of USD when multiple pairs
  fire pipeline simultaneously.
  Note: On restart, pending orders from pre-restart session
  are reconciled via snap_orders. Any cl_ord_ids found
  in snap_orders with pending status are added to registry.
  Assign: 1011002 WM-POR-001 through -007.

AMENDMENT TO AR-047 — Decimal.quantize() for all rounding:
  Replaces math.floor()/math.ceil() with Decimal.quantize().

  CONVERSION RULE (new BP-DEC-001):
    All numeric values received from Kraken (WS or REST):
    price: Decimal(str(msg["price"])) — NOT Decimal(float)
    qty:   Decimal(str(msg["qty"]))
    Apply to: ticker bid/ask, executions avg_price, order_qty,
    instrument price_increment/qty_increment/qty_min/cost_min.

  ROUNDING OPERATIONS (amendments to AR-047):
    order_qty:
      D_incr = Decimal(str(qty_increment))
      order_qty = raw_qty.quantize(D_incr, rounding=ROUND_DOWN)
    entry limit_price:
      D_incr = Decimal(str(price_increment))
      limit_price = raw_price.quantize(D_incr, rounding=ROUND_DOWN)
    TP limit_price:
      tp_price = raw_tp.quantize(D_incr, rounding=ROUND_UP)
    emergSL trigger price:
      sl_price = raw_sl.quantize(D_incr, rounding=ROUND_DOWN)

  Note: ROUND_DOWN/ROUND_UP are toward/away from zero.
  For positive prices: ROUND_DOWN = floor, ROUND_UP = ceiling.
  Use ROUND_DOWN and ROUND_UP (not ROUND_FLOOR/ROUND_CEILING)
  to avoid negative-price behavior (positive prices only).

  Add BP-DEC-001 through BP-DEC-005 to 1011001.
  Assign: 1011001 BP-DEC-001 through -005,
          1011002 WM-ROUND-007 through -012,
          0411001 amendment.
