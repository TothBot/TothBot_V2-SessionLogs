Change 5 — New static method _normalize_classic_key():
  Converts Kraken classic Ticker response key to altname form.
  Required because REST Ticker response uses full classic names
  (e.g. "XXBTZUSD") while instrument snapshot altname field
  uses shorter altnames (e.g. "XBTUSD").
  Strips Z-prefix from Z-prefixed fiat quotes (ZUSD→USD).
  Strips X-prefix from X-prefixed legacy crypto base (XXBT→XBT,
  XETH→ETH, XXRP→XRP).
  Known Z-fiats: ZUSD, ZEUR, ZGBP, ZCAD, ZJPY, ZCHF, ZAUD.
  Returns key unchanged if no match (handles modern pairs like
  SOLUSD which have no X/Z prefix issues).

2.1.3 Implementation Note — Altname vs Classic Name Gap
------------------------------------------------------------

The Kraken WS v2 instrument channel provides altname (e.g.
"XBTUSD") not the full classic name (e.g. "XXBTZUSD").
The REST Ticker response uses full classic names.

Resolution: _normalize_classic_key() converts response keys
to altname form before reverse map lookup. Tested against
known Kraken pair patterns:
  XXBTZUSD → XBTUSD   (strip leading X, strip Z from ZUSD)
  XETHZUSD → ETHUSD   (strip leading X after Z-fiat strip)
  XXRPZUSD → XRPUSD   (strip leading X, strip Z from ZUSD)
  SOLUSD   → SOLUSD   (no Z-fiat suffix — returned unchanged)

This correctly handles all current Kraken USD pairs.
If any pair misses (LIQUIDITY_PAIR_NOT_FOUND in log),
Gate 2 blocks entries for that pair (fail-safe, WM-LIQ-010).
