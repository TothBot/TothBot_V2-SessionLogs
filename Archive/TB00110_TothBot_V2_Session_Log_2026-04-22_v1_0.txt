3. SESSION ANOMALIES AND LESSONS

------------------------------------------------------------
3.1 PowerShell -> ssh -> bash quote escaping is fragile
------------------------------------------------------------

Observation: Compound ssh commands with embedded double
quotes (for JSON field matching like "event":"FATAL")
fail from PowerShell because the backslash-escape syntax
does not survive the multi-shell transit cleanly.
Symptom: "grep: Trailing backslash" error followed by
the && chain short-circuiting.

Lesson: during observation monitoring from Desktop
PowerShell, use either:
  (a) Individual ssh commands with plain-string grep
      (fast, slightly less precise)
  (b) Interactive SSH to VPS shell, then grep with
      single-quote wrapping (precise, extra step)
  (c) A one-time tothbot_check.sh script on disk,
      invoked via:  ssh root@<ip> "bash -s" < script

Did NOT escalate to OI because it is a Desktop tooling
artifact, not a system defect.

------------------------------------------------------------
3.2 PowerShell command concatenation risk
------------------------------------------------------------

Observation: C-4.12 and C-4.13 git add commands were
pasted on the same PowerShell line without Enter between
them. Git interpreted the concatenation as a single
command with `git` as a pathspec and failed with
"fatal: pathspec 'git' did not match any files".

Diagnostic: git status caught the state immediately.
No content corruption. All 4 intended adds were re-issued
cleanly one command at a time.

Lesson for TB00111 and beyond: strict adherence to
RULE 15 (one command per copy box). When supplying
commands, always put a visible gap/break between
commands so paste-without-Enter concatenation is
self-evident.
