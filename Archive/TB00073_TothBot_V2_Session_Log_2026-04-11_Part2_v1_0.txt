------------------------------------------------------------

VIOLATION 1 — 0311002 Phase 8 missing Code File Registry
maintenance rule.
  0311002 Phase 8 defined the coding sequence but did not
  include a mandatory rule requiring the Code File Registry
  in 1010000 to be updated after every code session.
  Without this rule, the governance gap was not explicitly
  prohibited — it was simply unspecified. The corrective
  action closes this gap.

Phase 8 completion.
  1010000 dv1_0 was written at Phase 8 start (04-05-2026)
  with status "to be written" or "WD dv1_X (as of Phase 8
  start)" for all 14 instance documents. Phase 8 was
  completed on the same date — all 14 specs reached
  CODING-READY state — but 1010000 was never updated to
  reflect completion. Every subsequent session operated
  with a stale anchor document.

VIOLATION 3 — 1010000 missing Code File Registry section.
  No section in 1010000 formally mapped .py files to their
  governing specs. This section was required from the moment
  any .py file was first written. It was never created.
  This is the primary document control gap: without this
  registry, there is no formal traceability between code
  and the document control system.

VIOLATION 4 — ws_manager.py header references dv1_12
instead of dv1_13.
  ws_manager.py was updated this session to fix
  DEFECT-WM-OHLC-001 (spec documented in dv1_13) but the
  in-file header was not updated to reference dv1_13.
  All other .py file headers correctly reference their
  current governing spec version.

VIOLATION 5 — Project folder contained stale superseded
document versions alongside current versions.
  Per VP-010, superseded versions are managed through git
  history. The project folder should contain only the
