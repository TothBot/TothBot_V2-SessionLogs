GOVERNED BY:  1011002 WS_Manager_Coding_Spec

Three call sites are unsigned:

  (a) _rest_get_ws_token    (approx lines 602-629)
      Endpoint: /0/private/GetWebSocketsToken
      Retrieves trade_secret but does not use it.

  (b) _rest_get_open_orders (approx lines 656-666)
      Endpoint: /0/private/OpenOrders
      Does not retrieve secret at all.

  (c) _rest_get_account_balance (approx lines 668-678)
      Endpoint: /0/private/Balance
      Does not retrieve secret at all.

All three ship only API-Key + nonce. Kraken correctly
rejects with EAPI:Invalid key. The error text is
misleading - keys are valid; requests are unsigned.

SIGNING CONTRACT (0411002 dv1_1 REST-AUTH-001):

  message   = URI_path + SHA256(nonce + POST_data_string)
  signature = HMAC-SHA512(message, base64_decode(api_secret))
  API-Sign  = base64_encode(signature)

REQUIRED HEADERS (0411002 dv1_1 REST-AUTH-003):

  API-Key:      TRADE API key value (or DATA key for
                DATA-scoped endpoints)
  API-Sign:     HMAC-SHA512 signature (above)
  Content-Type: application/x-www-form-urlencoded

RECOMMENDED PART 2 IMPLEMENTATION:

Add a private helper on WSManager:

  def _sign_private_request(
      self,
      uri_path: str,
      postdata: dict,
      api_secret: str,
  ) -> tuple[dict, str]:
      """
      Returns (encoded_postdata, api_sign_header).
      Per 0411002 REST-AUTH-001.
      """
      import hashlib, hmac, base64
      from urllib.parse import urlencode

      post_str = urlencode(postdata)
      sha256_digest = hashlib.sha256(
          (postdata["nonce"] + post_str).encode()
      ).digest()
      message = uri_path.encode() + sha256_digest
      mac = hmac.new(
          base64.b64decode(api_secret),
          message,
          hashlib.sha512,
      )
      api_sign = base64.b64encode(mac.digest()).decode()
      return post_str, api_sign

Apply at all three call sites. Pass Content-Type:
application/x-www-form-urlencoded and the urlencoded
post_str in the request body.

FP/DP NOTE: The helper encapsulates the signing
contract in one place. If Kraken changes the
algorithm (REST-AUTH-002 notes the Oct 2025 change
applied to Futures only; Spot unchanged), only the
helper changes.
