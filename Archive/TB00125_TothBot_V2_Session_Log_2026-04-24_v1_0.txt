2.3 Part 2 authoring (Claude Chat) - three DC files
------------------------------------------------------------

DEFECT-WM-REST-SIGN-001 root cause (from TB00120 Session
Log v1_1 §2.6):

  Three private Kraken REST call sites in ws_manager.py
  shipped only the API-Key header + nonce. No API-Sign
  HMAC-SHA512 signature attached. Kraken correctly
  rejected every such request with EAPI:Invalid key per
  0411002 dv1_1 REST-AUTH-001 / REST-AUTH-003. Paper-
  mode impact zero (WM-RECONNECT-019 / FIX-WM-PT-001
  skip branches gate all three sites to no-op). Live-
  mode transition would have failed 100%.

  Site 1: _rest_get_ws_token
          endpoint /0/private/GetWebSocketsToken
          dv1_22 lines 639-666. Retrieved trade_secret
          but orphaned it; never used.

  Site 2: _rest_get_open_orders
          endpoint /0/private/OpenOrders
          dv1_22 lines 693-703. Did not retrieve secret.

  Site 3: _rest_get_account_balance
          endpoint /0/private/Balance
          dv1_22 lines 705-715. Did not retrieve secret.

Fix design (authored and validated in Claude Chat):

  New private helper on WSManager:

    _sign_rest_request(url_path: str,
                       data: dict,
                       api_secret: str,
                      ) -> tuple[str, str]

  Implements the 0411002 REST-AUTH-001 contract:

    message   = URI_path + SHA256(nonce +
                                  POST_data_string)
    signature = HMAC-SHA512(message,
                            base64_decode(api_secret))
    API-Sign  = base64_encode(signature)

  Returns (post_data_string, signature_b64). Helper is
  the SOLE place in the code base where Kraken REST
  signing is implemented; all call sites invoke it.

  Algorithm validated byte-exact against Kraken's
  published test vector (AddOrder example,
  nonce=1616492376594, pair=XBTUSD, price=37500,
  volume=1.25) - computed signature matched expected
  signature bit-for-bit.

  All three call sites rewritten verbatim to follow the
  mandated pattern: retrieve both key and secret, build
  data dict with nonce, invoke _sign_rest_request, POST
  with headers {API-Key, API-Sign, Content-Type} and
  body = post_data_string. The signed bytes and
  transmitted bytes are byte-identical by construction
  (WM-REST-SIGN-003 requirement).

  Imports added: base64, hashlib, hmac,
  urllib.parse.urlencode. No other behavior changed.
  Paper-mode skip branches untouched.
