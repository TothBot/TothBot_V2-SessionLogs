  Original singular-CIATS purpose statement preserved
  in opening paragraph. New paragraph appended codifying
  per-module instantiation: each trading module
  (mod:Long_Module, mod:Short_Module, future modules)
  instantiates its own CIATS instance with its own
  statistical pool + EWMA monitor + Trade Outcome Bus
  subscription + statistical-method state + PDCA cycle
  state + Proposal Engine + Parameter Store namespace +
  Regime Library. Module identifier is the routing key.
  Cross-module data contamination prohibited.

3.3 SECTION 3 FIVE CODING-READINESS QUESTIONS — PER-
    INSTANCE REFRAMING.

  All five questions reframed for per-instance behavior:
    (1) Each per-module CIATS instance reads filtered
        Logger output and operates two streams per
        instance.
    (2) Interfaces filtered by module identifier; each
        instance writes only to its own module's
        Parameter Store namespace.
    (3) Hard rules extended with per-instance 200-trade
        floor + per-instance 600-trade per-regime floor
        + cross-module contamination prohibition +
        Half-Kelly reframing per TB00000 v2_29 §9.17
        retirement + per-module Kelly normalization rule.
    (4) Triggers operate per instance.
    (5) Every log entry carries module= field.

3.4 SECTION 4 STATISTICAL THRESHOLDS — PER-INSTANCE.

  CIATS-ST-001 / CIATS-ST-002 / CIATS-ST-003 reframed to
  per-instance thresholds. Each instance evaluates its
  own threshold against its own statistical pool. Cross-
  module threshold aggregation prohibited. CIATS-ST-002
  Half-Kelly language updated to "CIATS may propose Half-
  Kelly (or another sizing-curve methodology) under PDCA
  cycle" per TB00000 v2_29 §9.17 retirement of hardcoded
  transition.

3.5 SECTION 5 STATISTICAL METHODS — PER-INSTANCE
    APPLICATION.

  Methods (EWMA, CUSUM, Mann-Whitney U, Spearman, Kelly)
  remain mechanically identical but now explicitly
  applied per instance on instance-own data. Cross-
  instance statistical aggregation prohibited.

3.6 SECTION 5.5 HALF-KELLY — REFRAMED AS CANDIDATE
    METHODOLOGY (COLLATERAL AMENDMENT).

  Per TB00000 v2_29 §9.17 RETIRED AT v2_29: "Hardcoded
  Half-Kelly transition at 200 trades — Half-Kelly
  remains a candidate sizing methodology CIATS may adopt
  as the underlying sizing-curve form. It is no longer a
  hardcoded transition rule."

  Section 5.5 reframed: Half-Kelly is a CANDIDATE sizing-
  curve methodology each per-module CIATS instance may
  propose under PDCA cycle. CIATS-KE-001 through
  CIATS-KE-007 updated:
    - CIATS-KE-001: formula unchanged but explicitly
      per-instance.
    - CIATS-KE-002: W and R drawn from instance-own
      module-tagged trade pool only.
    - CIATS-KE-003: NORMALIZATION REWRITTEN. Retired
      tradeable_pct / max_concurrent formula REPLACED
      by per_trade_ceiling_pct[module] ×
      module_wallet_balance_usd[module]. Applied in
      dollars (not percent of abstract portfolio).
    - CIATS-KE-004 through -006: per-instance with
      module identifier in log output.
    - CIATS-KE-007: cross-reference unchanged.

  Collateral amendment required for per-module
  instantiation internal consistency. The retired

  max_concurrent) cannot be referenced in a post-v2_29
  spec.

    CIATS-ISO-005.

  per-module CIATS instance operates under this isolation
  contract independently" annotation.

  EWMA state.

3.9 SECTION 6.3 STREAM 2 TRADE OUTCOME BUS — MODULE-
    ROUTED + NEW CIATS-TOB-004.

  CIATS-TOB-001 / 002 / 003 reframed for per-instance

  filtering at consumption boundary.

  CIATS-TOB-004 NEW: TRADE OUTCOME BUS ROUTING BY
  MODULE IDENTIFIER. Every TRADE_CLOSE event MUST carry

  filters subscription on this field. Non-matching
  events IGNORED. Unidentified or unrecognized-
  identifier events logged as CIATS_EVENT_ROUTING_ERROR
  at WARNING level and dropped.
