2.2 Pre-Existing Defects Discovered — CRITICAL
------------------------------------------------------------

Code audit of the VPS repo revealed multiple critical defects
that will cause TothBot to crash on the first trade attempt.
These defects were present before this session and were NOT
caught by the 95 unit tests (unit tests mock these
interactions).

DEFECT-001: execution_engine.py — create_record crash
  Location:  tothbot/execution_engine.py line 255
  Code:      self._pm.create_record(symbol, cl_ord_id,
                                    entry_limit_price, entry_qty)
  Problem:   PositionMirror class has no create_record() method.
             The method is named create() with a completely
             different signature.
  Impact:    AttributeError crash on EVERY entry dispatch.
             TothBot cannot execute a single trade.
  Severity:  CRITICAL — blocks all paper trading.

DEFECT-002: execution_engine.py — clear_record crash
  Location:  tothbot/execution_engine.py lines 505 and 538
  Code:      self._pm.clear_record(symbol)
  Problem:   PositionMirror class has no clear_record() method.
  Impact:    AttributeError crash on every entry cancel or
             GTD expiry with no fill.
  Severity:  CRITICAL.

DEFECT-003: execution_engine.py — pair_specs does not exist
  Location:  tothbot/execution_engine.py line ~198
  Code:      pair_spec = self._wm.pair_specs.get(symbol, {})
  Problem:   WSManager attribute is pair_cache, not pair_specs.
  Impact:    AttributeError crash before any order is sent.
  Severity:  CRITICAL — blocks all paper trading.

DEFECT-004: WSManager._handle_filled — wrong event format
  Location:  tothbot/ws_manager.py _handle_filled() method
  Code:      await self._exec_engine_fn(
                 {"symbol": symbol, "entry_fill_price": avg_price,
                  "qty": cum_qty, "fees_entry_usd": fees}, self)
  Problem:   Event dict has no cl_ord_id or exec_type field.
             LongModule.on_execution_event() looks for cl_ord_id
             to route to EE. Gets "" → ctx is None → returns.
             EE._on_entry_filled() is never called.
  Impact:    TP and emergSL orders are NEVER placed after any
             entry fill. Every open position has NO protection.
  Severity:  CRITICAL — catastrophic risk to open positions.

DEFECT-005: batch_add ACK never handled
  Location:  tothbot/ws_manager.py _channel_dispatch dict
  Problem:   Kraken WS v2 batch_add ACK returns with
             method="batch_add". This is not in _channel_dispatch.
             WSManager logs UNKNOWN_MESSAGE_TYPE and discards it.
             EE.on_batch_add_response() is never called.
  Impact:    wm.position_mirror[symbol].tp_order_id and
             emergsl_order_id remain empty string on EVERY fill.
             ExitController cancel calls use empty string order IDs.
             Layer 1a, 1b, and Layer 2 exits cannot cancel orders.
  Severity:  CRITICAL.

DEFECT-006: RiskEngine MTM uses wrong position_mirror
  Location:  tothbot/risk_engine.py lines 205, 254
  Code:      self._pm.all_records  and  self._pm.open_count
  Problem:   self._pm is the standalone PositionMirror object.
             Runtime positions are in wm.position_mirror (dict).
             PositionMirror._mirror is never populated at runtime
             because EE's create_record call crashes (DEFECT-001).
  Impact:    Drawdown MTM is always 0 regardless of open
             positions. Circuit breakers do not fire correctly.
  Severity:  HIGH — affects loss prevention.

DEFECT-007: test_integration.py never committed to GitHub
  Evidence:  git log --all --full-history -- tests/test_integration.py
             returned nothing. File does not exist in any commit.
  Impact:    The 8 integration tests from TB00052 exist only in
             Claude session history. They are not in the repo.
             Phase 10 completion claim of 103/103 tests is
             incorrect. Actual: 95 unit tests only.
  Severity:  HIGH — Phase 10 must be retroactively corrected.
