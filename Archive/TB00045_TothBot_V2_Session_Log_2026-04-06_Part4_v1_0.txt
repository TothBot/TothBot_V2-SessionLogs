  D-LM-001  LongModule is the PRIMARY execution event router.
            WS Manager routes executions channel events to
            lm.on_execution_event() only. LM calls
            ee.on_execution_event() for cases requiring EE
            order construction. LM handles exec_type=new and
            exec_type=trade exclusively (EE ignores both).

  D-LM-002  Below-minimum partial fill (AR-054 CASE B):
            LM pops ee._entry_orders[cl_ord_id] before
            delegating, preventing EE from attempting invalid
            batch_add on below-minimum qty. LM issues IOC
            market sell, clears PM, releases semaphore.

  D-LM-003  BoundedSemaphore acquired in on_gate8_output()
            BEFORE calling EE. EE calls release_semaphore()
            internally on size-below-min failure path.
            LM on_position_closed() releases semaphore on
            normal position close.

  D-LM-004  Gate 8 context (signal_params, asset_regime,
            market_regime) stored in _gate8_context[symbol]
            at dispatch. EE does not carry this data.
            POSITION_OPENED log uses this context on fill.

  D-SC-001  SC-GATE-3 uses symbol in self._pm (dict protocol,
            O(1)) -- consistent with how exit_controller.py
            accesses wm.position_mirror.

  D-CIATS-001  CIATS reads Logger files via poll/tail (30s
               interval). Also exposes on_trade_close() for
               same-cycle processing from Exit Controller.
               Log file remains authoritative source.
