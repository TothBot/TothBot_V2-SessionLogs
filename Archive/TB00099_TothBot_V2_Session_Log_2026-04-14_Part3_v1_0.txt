2.2 DEFECT-SS-008 — Root Cause Analysis
------------------------------------------------------------

Root cause identified via expanded grep and code inspection.

DEFECT-SS-008: quote_currency derived from wrong source.

_handle_instrument() populates pair_cache with:
  "quote_currency": pair.get("quote_currency", "")

Kraken WS v2 instrument snapshot does NOT provide a
"quote_currency" field. pair.get("quote_currency", "")
ALWAYS returns "".

Downstream consequences:

  (a) _build_monitored_universe():
        quote = spec.get("quote_currency", "")
        if quote not in ("USD", "USDC"): continue
      ALL pairs fail this filter. candidates = [].
      Only BTC/USD survives via hardcoded fallback.
      → monitored_universe = ["BTC/USD"] (pairs=1).

  (b) _warm_up_all_pairs():
        candidates = [sym for sym, spec in pair_cache...
          if spec.get("quote_currency") in ("USD","USDC")
          and spec.get("status") == "online"]
      Same filter. candidates = [] always.
      → _rest_get_ticker([]) → {} immediately (silent).
      → Both startup attempts return {}.
      → LIQUIDITY_SEED_FAILED.
      → Gate 2 blocks all entries. Zero trading.

The PAIR_MAPS_BUILT=1498 in dv1_17 was correct — but the
map build is irrelevant because candidates is always [].
The symptom LIQUIDITY_SEED_FAILED persisting after dv1_17
was therefore expected and correct — dv1_17 only fixed the
map source, not the empty candidates root cause.

Pre-existing spec defect also identified:
WM-PS-005 through WM-PS-009 in Section 10 of 1011002
spec were incorrectly prefixed WM-PS (same prefix as
Pair Status rules in Section 9), creating a rule number
collision. Fixed in 1011002 dv1_17 by renaming to
WM-SZ-001 through WM-SZ-005.

2.3 DEFECT-SS-008 Fix — Delivered
------------------------------------------------------------

Two documents produced with full document control.

DOCUMENT 1: ws_manager.py dv1_18

  Single code change. _handle_instrument() line 1207.

  OLD (wrong):
    "quote_currency": pair.get("quote_currency", ""),

  NEW (correct):
    "quote_currency": symbol.split("/")[1] if "/" in symbol else "",

  All Kraken WS v2 symbols are "BASE/QUOTE" format.
  The quote asset is ALWAYS the right side of "/".
  "BTC/USD".split("/")[1] → "USD"
  "ETH/USDC".split("/")[1] → "USDC"
  No dependency on instrument snapshot fields.

  Additional changes:
    pair_cache comment updated to include quote_currency
    and note the derivation rule (WM-PS-006, DEFECT-SS-008).

  Governed by 1011002 dv1_17 WM-PS-006.
