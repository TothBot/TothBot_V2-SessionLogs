2.3 Root Cause Analysis — DEFECT-SS-004
------------------------------------------------------------

Code inspection confirmed the following:

In ws_manager.py, paper mode startup path
(_startup_connect_and_subscribe):

  1. Sets paper balance ($10,000)
  2. Connects public WS
  3. Calls _subscribe_public()
  4. Resets zombie timer
  5. Returns immediately ← THE DEFECT

In _main_loop(), paper mode path runs:
  - _recv_loop_public()
  - _ping_loop()
  - _zombie_monitor()

REST warm-up (Step 8 per SS-WARMUP-001 through -008 in
1011014) is NEVER called in paper mode. Specifically:
  - GetOHLCData(interval=5) never called → ATR not seeded
  - GetOHLCData(interval=60) never called → EMA not seeded
  - GetOHLCData(interval=1440) never called → Regime unknown
  - No pair ever transitions WARM_UP → READY
  - Signal pipeline never evaluates a candle
  - VPS_STARTUP_COMPLETE never logged
  - sd_notify READY=1 never sent to systemd

The system receives live OHLC candles from Kraken WS.
Every candle is silently dropped because BTC/USD is
permanently in WARM_UP state. The pipeline gate that
checks pair status (pre-Gate 1) rejects every candle.
This has been happening since 03:03 UTC. Paper trading
has never executed a single pipeline evaluation.

Secondary observation:
  OHLC interval=60 subscription is rejected by Kraken:
  "Already subscribed to one ohlc interval on this symbol"
  This is known issue PL-NEW-011. It is a separate defect
  from DEFECT-SS-004. PL-NEW-011 is on the Parking Lot.
  It does not block the fix for DEFECT-SS-004 but must
  be addressed in the same fix session as the 1H HTF
  cache is not populating via WS either.
