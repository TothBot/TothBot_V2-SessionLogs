FINDING 6 (CRITICAL) — Paper synthetic ledger never
  updated on simulated fills.

  Spec requirement (TB00000 v2_18 §4.4 PA-004 #3):
    "synthetic spot_usd_balance starting at
     paper_starting_balance ($1,000 per D-05); updated
     by simulated-fill P&L applied locally using the
     SAME fee schedule."

  Code reality at fb5de32:
    spot_usd_balance is set ONCE in paper mode at
    _startup() and never updated by simulated fills.
    Only update path is _handle_balances() which is
    live-only by HR-WM-022.

  Behavioral consequences:
    - Drawdown computation broken after first close.
      _compute_drawdown_for_pair computes
      current = spot_usd_balance + open-position MTM.
      In live, spot_usd_balance accumulates closed-
      trade cash. In paper it doesn't. Drawdown
      numerator wrong.
    - CIATS Trade Outcome Bus receives only PositionRecord
      (entry_price); exit_price from _maybe_paper_fill
      is logged but NOT transmitted to CIATS bus. CIATS
      cannot reconstruct paper P&L.
    - PA-001 violation. Paper ledger does not maintain
      a state that mirrors live's balances-channel
      ledger evolution.

  Severity: CRITICAL. Drawdown circuit breakers fire on
  wrong numbers. CIATS data invalid.

  Exit Controller entirely.

  Spec requirement (0511001 dv1_4 HR-WM-025):
    "Paper mode does not affect OHLC processing,
     indicator updates, pipeline triggering, or CIATS
     data flows. These operate identically in both
     modes."

  Spec requirement (TB00000 v2_18 §4.4 PA-005):
    "Outside the four PA-004 divergence points, paper
     and live MUST be byte-identical."

  Code reality at fb5de32:
    Live path:  _handle_filled → dispatch_market_sell or
                EE callback → exit_controller._close_position
                → TRADE_CLOSE log emitted with full P/L
                payload.

    Paper path: _maybe_paper_fill detects TP/SL →
                position_mirror.pop(symbol, None) directly
                → _fire_ciats_outcome called with
                (symbol, exit_reason, pos) signature →
                no exit_price, no fees, no realized P/L
                in payload → NO TRADE_CLOSE log emitted.

  Behavioral consequences:
    - CIATS Half-Kelly polls TRADE_CLOSE log records.
      Paper produces zero TRADE_CLOSE records. CIATS
      cannot operate on paper data. PDCA broken in
      paper mode.
    - Position close handling is NOT in PA-004 four
      divergence points. The bypass is a defect, not a
      permitted divergence.

  Severity: CRITICAL. Paper-as-master architecturally
  broken at the close-path layer.
