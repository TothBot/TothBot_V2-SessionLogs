2.2 DEFECT-WM-ZOM-001 — Zombie Timer Starts at __init__
------------------------------------------------------------

Symptom:
  ZOMBIE_CONNECTION_DETECTED CRITICAL at elapsed_seconds=
  99.9 on every service instance.

Root cause:
  _last_real_data_public initialized to time.monotonic()
  at WSManager.__init__(). Construction occurs before
  wm.run() is called. Startup overhead consumed the
  90-second zombie budget before the recv loop started.

Fix:
  Reset _last_real_data_public = time.monotonic() at END
  of _startup_connect_and_subscribe() in both paper mode
  and live mode paths. Clock now starts from connection-
  live, not construction.

Commit: 4241bf3 — deployed to VPS. Zombie still fired.
DEFECT-WM-OHLC-001 was primary root cause (Section 2.3).

------------------------------------------------------------
2.3 DEFECT-WM-OHLC-001 — ohlc Subscription Without Symbols
------------------------------------------------------------

Symptom:
  Zombie still fired at elapsed_seconds=99.9 after
  DEFECT-WM-ZOM-001 fix.

Root cause confirmed from full log inspection:
  _subscribe_public() sent four subscriptions at startup:
    instrument  — SUCCESS (INSTRUMENT_SNAPSHOT_COMPLETE,
                  1506 pairs)
    status      — FAIL: "Symbol(s) not found"
    ohlc(5)     — FAIL: "Symbol(s) not found"
    ohlc(60)    — FAIL: "Symbol(s) not found"

  Kraken WS v2 ohlc channel requires a symbol list.
  Without it Kraken returns "Symbol(s) not found".
  Without ohlc data, _last_real_data_public was only
  updated by the initial instrument snapshot and the
  auto-sent status message on connect — both one-time
  events. 90 seconds later zombie fired. Every time.

Fix:
  (1) Removed ohlc(5) and ohlc(60) from
      _subscribe_public(). These cannot be sent at
      startup — monitored_universe not yet built.
  (2) Added _subscribe_ohlc_channels() method. Sends
      ohlc(5) and ohlc(60) with symbol=monitored_universe.
      Logs OHLC_SUBSCRIBED (INFO) on success.
      Logs OHLC_SUBSCRIBE_SKIPPED (WARN) if empty.
  (3) Called via asyncio.create_task() from
      _handle_instrument() after msg_type=="snapshot"
      and _build_monitored_universe() completes.

Status: Fixed in ws_manager.py. NOT YET committed to
GitHub or deployed to VPS.
