2.2 Defect Found and Corrected -- 1011010 dv1_5
------------------------------------------------------------

While writing 0211001 CIATS_Specification, a cross-document
inconsistency was identified in CIATS-CU-001:

  DEFECT: 1011010 dv1_4 specified CUSUM h=5.0 (unitless
  absolute value -- dimensionally wrong).

  CORRECT: All approved documents specify h=4sigma (relative
  to the monitored metric's standard deviation):
    0500000 dv1_18: "CUSUM parameters (dv1_6): k=0.5*sigma,
                     h=4*sigma. Starting values."
    0200000 v1_0:   "CUSUM: k=0.5sigma, h=4sigma."
    0311002 v1_0:   "h = 4sigma of the monitored metric."

  FP/DP RATIONALE for h=4sigma vs h=5sigma:
    h=4sigma: ARL_0 ≈ 168, ARL_1 ≈ 10.4 for 1sigma shift.
    h=5sigma: ARL_0 ≈ 465, ARL_1 ≈ 14.3 for 1sigma shift.
    h=4sigma is loss-minimization correct: faster drift
    detection (ARL_1 ≈ 10 vs 14) catches parameter
    degradation sooner. False alarms filtered downstream
    by Spearman (|rho|>0.3, p<0.05) and Mann-Whitney
    (alpha=0.01) gates before any parameter changes.
    h is dimensionless multiplier of metric sigma -- not
    an absolute value. h=5.0 absolute is wrong.

  CORRECTION: 1011010 dv1_5 written. CIATS-CU-001 corrected
  to h=4sigma. Section 3.12 added with full FP/DP rationale,
  ARL values, three reasons h=4sigma is correct for TothBot,
  and CUSUM implementation pseudocode. dv1_4 removed from
  GitHub. dv1_5 committed.

  IMPACT: Only 1011010 required correction. 0500000, 0200000,
  0311002, and 0211001 all already correctly specify h=4sigma.
