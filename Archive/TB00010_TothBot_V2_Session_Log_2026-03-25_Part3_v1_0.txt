3.3 New Gaps and Improvements Found (ALL RESOLVED)
------------------------------------------------------------

GAP-B (dv1_10) ★ CRITICAL — stp_type Format Correction:
  FINDING: Official Kraken WS v2 documentation and batch_add
  example uses underscore format for stp_type values:
    "stp_type": "cancel_both"   (underscore — WS v2 correct)
    "stp_type": "cancel_newest" (underscore — WS v2 correct)
  REST API uses hyphen format: cancel-newest, cancel-both.
  ALL prior TothBot V2 specifications used "cancel-newest"
  (hyphen) — this is the REST format, WRONG for WS v2.
  Using hyphen format in WS v2 will cause order rejection
  or silent STP failure. This is a critical error in all
  prior specifications dv1_0 through dv1_9.
  Resolved in: 0500000 dv1_10 (all text + SVG diagrams),
               0411001 dv1_3 WS-STP-001 through -003,
               1011002 dv1_3 WM-STP-001 through -003,
               1011001 dv1_1 BP-ORD-001, BP-TEST-004.
  AR-032 added.

A-21 (dv1_10) ★ — deadline Parameter on add_order / batch_add:
  FINDING: Both WS v2 add_order and batch_add support a
  deadline parameter (ISO 8601 UTC timestamp). Engine
  rejects orders arriving after this time. Provides
  protection against stale order placement on network delay.
  Range: 500ms–60s. Default if omitted: 5s.
  Standard: deadline = now + 5s.
  Resolved in: 0500000 dv1_10 A-21, AR-033,
               0411001 dv1_3 WS-DL-001 through -004,
               1011002 dv1_3 WM-DL-001 through -004.

A-22 (dv1_10) ★ — aiohttp ClientTimeout + TCPConnector:
  FINDING: aiohttp.ClientSession without explicit timeout
  can hang indefinitely on degraded REST servers. This
  blocks reconnect/gap recovery sequences.
  ClientTimeout(total=10, connect=5, sock_read=8) +
  TCPConnector(limit=10, limit_per_host=5, force_close=False).
  force_close=False enables HTTP keepalive.
  Resolved in: 0500000 dv1_10 A-22, AR-035,
               0411001 dv1_3 WS-REST-005 through -008,
               1011002 dv1_3 WM-REST-006 through -011,
               1011001 dv1_1 BP-HTTP-001 through -004.

A-23 (dv1_10) ★ — Ticker event_trigger:bbo for Open Positions:
  FINDING: Ticker event_trigger:bbo fires on every bid/ask
  change. For long positions, bid is the realizable exit
  price. bbo provides faster MAE detection than trades-only.
  For pairs without open positions, trades trigger is
  sufficient and generates less volume.
  WS Manager must manage per-pair trigger based on Position
  Mirror state, resubscribing when positions open/close.
  Resolved in: 0500000 dv1_10 A-23, AR-034,
               0411001 dv1_3 WS-TKR-001 through -004,
               1011002 dv1_3 WM-TKR-001 through -004.

A-24 (dv1_10) ★ — Regime Engine REST Rate Limit Stagger:
  FINDING: Kraken public REST OHLC endpoint is rate-limited
  by IP + currency pair at ~1 call/second. Calling for
  50+ pairs simultaneously at 00:00 UTC triggers HTTP 429
  responses and corrupts regime classification.
  asyncio.sleep(1.1) between pair calls required.
  50 pairs × 1.1s = 55s total. Completes before first candle.
  Resolved in: 0500000 dv1_10 A-24, AR-036,
               0411001 dv1_3 WS-OHLC-006 through -009,
               1011002 dv1_3 WM-RL-004,
               1011012 dv1_1 RE-RL-001 through -004.

AR-037 — API Key IP Whitelisting:
  FINDING: Kraken supports per-key IP whitelisting for both
  DATA and TRADE keys. VPS has fixed IP — ideal for this.
  Both keys must be whitelisted to VPS IP. Must be updated
  on any VPS migration (before migration, not after).
  Direct Kraken AWS PoP access is NOT available for standard
  Spot trading — Hillsboro migration remains the correct
  latency optimization.
  Resolved in: 0500000 dv1_10 AR-037,
               0411001 dv1_3 WS-KEY-006 through -009,
               1011013 dv1_1 VD-KEY-006 through -009.
