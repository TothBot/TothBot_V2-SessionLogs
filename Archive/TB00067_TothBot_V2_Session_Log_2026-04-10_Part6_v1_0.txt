2.3 Root Cause Analysis — Full Code Deep Dive
------------------------------------------------------------

Full ws_manager.py code audit performed.

Root cause confirmed: _startup_connect_and_subscribe() calls
_rest_get_ws_token() unconditionally at line 590 of the
pre-paper version (same line in paper trading version).
No paper_mode gate exists on this call.

In paper trading mode:
  - Order dispatch is suppressed (add_order, batch_add,
    cancel_order, amend_order, batch_cancel,
    dispatch_market_sell all have paper_mode gates)
  - Fills are simulated via _simulate_entry_fill()
  - Paper TP/emergSL detection via ticker handler
  - BUT: the private WS is still connected at startup
  - AND: GetWebSocketsToken is still called at startup
  - AND: _recv_loop_private() is still started
  - AND: _ping_loop() still pings private WS
  - AND: _zombie_monitor() still monitors private WS

The TRADE API key on VPS is either:
  (a) Invalid / expired
  (b) Not set or set incorrectly in .tothbot.env

In paper mode, the TRADE API key is irrelevant — no real
orders are dispatched. The private WS should not be
connected in paper mode at all.
