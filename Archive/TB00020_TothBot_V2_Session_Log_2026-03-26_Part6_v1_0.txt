FINDING 7: GAP-B — SC State Management — NEW, RESOLVED
------------------------------------------------------------

SC-GATE-4 and SC-GATE-6 require in-memory state dicts.
These were not previously specified in WS Manager.

RESOLUTION:
WS Manager maintains two per-symbol in-memory dicts:
  exit_cooldown_log: dict[str, float]
    symbol → time.monotonic() at last position close
  consecutive_loss_count: dict[str, int]
    symbol → consecutive losses without a win

Update rules on position close:
  Always: exit_cooldown_log[symbol] = time.monotonic()
  On loss exit: consecutive_loss_count[symbol] += 1
  On TP_FILL (win): consecutive_loss_count[symbol] = 0

Loss exits: MAE_THRESHOLD_BREACH, TIME_EXPIRY,
  HTF_REGIME_REVERSAL, DAILY_REGIME_DOWNGRADE, SIGNAL_DECAY.

Both dicts PRESERVED across reconnect (same principle as
portfolio_baseline_USD per AR-052/AR-056).
