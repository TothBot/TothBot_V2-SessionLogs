2.2 Root Cause Analysis — DEFECT-SS-009
------------------------------------------------------------

DEFECT-SS-009 (CRITICAL — CONFIRMED AND FIXED)

Root cause: _regime_wrapper in startup_sequence.py called
  rge.run_daily_computation() with no universe argument.
  run_daily_computation() uses self._universe when no
  universe is passed. rge._universe is set at RegimeEngine
  construction time from _load_universe(config).
  _load_universe() returns ["BTC/USD"] only (TOTHBOT_UNIVERSE
  env var not set; default is "BTC/USD").

  The full 50-pair monitored universe is built by WSManager
  at runtime from the Kraken instrument snapshot — it is NOT
  available at construction time. rge._universe = ["BTC/USD"]
  for the lifetime of the process.

  Result: regime computed for 1 pair (BTC/USD) only.
  All 49 other monitored pairs had NO_REGIME_DATA.
  Gate 3 blocked all non-BTC entries for the entire session.
  This also affects the daily 00:00 UTC regime refresh —
  same wrapper, same defect.

  Evidence:
    REGIME_STARTUP_SEED_COMPLETE in ~1.25s (BTC/USD: 1.1s
    stagger + ~0.15s REST fetch = ~1.25s correct for 1 pair).
    All non-BTC pairs showing GATE_3_FAIL NO_REGIME_DATA
    27 minutes after SYSTEM_OPERATIONAL.

Fix: _regime_wrapper now passes list(wm.monitored_universe)
  to rge.run_daily_computation(universe=universe). Python
  closures capture wm by reference — at call time,
  wm.monitored_universe contains the full ranked 50-pair
  universe. Guard: if wm.monitored_universe else None.
