3. PRIORITY LEGEND
============================================================

CRITICAL  Architecturally load-bearing; diagram or
          prose carries an out-of-date or contradicted
          claim that would mislead a reader from the
          current source-of-truth in 0500000 dv1_24
          structured tags + Phase 1 close-gate specs.

HIGH      Diagram or prose omits a now-locked
          architectural element (LOCKED decision,
          new HR-rule, new PT-MODE rule, new AR rule)
          but does not actively contradict.

MEDIUM    Diagram or prose mismatch is cosmetic /
          structural-tagging incomplete / element
          count mismatch / label drift.

LOW       Documentation hygiene; not directly diagram-
          relevant but cited in defect chain.

============================================================
4. DISPOSITION LEGEND
============================================================

ALTER       Element exists in current diagram but must
            be edited (label change, R-increment,
            constraint restoration, count fix).
ADD         Element does not exist in current diagram
            but must be introduced.
REMOVE      Element exists in current diagram but is
            no longer architecturally valid and must
            be retired.
NO ACTION   Surfaced in session log but already closed
            elsewhere (governance, code-deploy,
            superseded by later cascade) with no
            residual diagram or prose impact.
DEFERRED    Closure path identified but execution out
            of TB00154+ diagram-rework scope.
