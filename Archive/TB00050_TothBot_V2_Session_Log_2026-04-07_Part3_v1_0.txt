2.1 FP/DP Analysis — Position Mirror Write Ownership
------------------------------------------------------------

Full audit performed across all relevant documents and
source files. Zero memory used. Documents read:

  0511005  Position_Mirror_Specification dv1_0       (full)
  0511003  Execution_Engine_Specification dv1_0      (full)
  1011006  Position_Mirror_Coding_Spec dv1_0         (full)

  exit_controller.py                                 (partial)
  signal_pipeline.py                                 (partial)
  risk_engine.py                                     (partial)
  long_module.py                                     (partial)

Key findings:

  (a) OI-001: PositionRecord field name mismatch.
      1011006 spec used: cl_ord_id_entry, emgsl_order_id,
      emgsl_cl_ord_id, and was missing entry_limit_price.
      ws_manager.py actual: cl_ord_id, emergsl_order_id,
      emergsl_cl_ord_id. exit_controller.py consistent
      with ws_manager.py actual names.
      Spec was wrong. Code was internally consistent.

  (b) OI-002: ExecutionEngine._pm methods non-existent.
      EE called self._pm.create_record(), on_entry_filled(),
      on_batch_add_ack(), clear_record() — none existed
      anywhere. Dead code. Crash at runtime.

  (c) Architectural finding: HR-PM-009 and 0511003 §7.4
      both state WSManager is SOLE WRITER to position_mirror.
      EE providing a separate PositionMirror object violates
      this rule. Correct fix: PM write methods on WSManager;
      EE calls self._wm.pm_*().

  (d) Additional finding: position_mirror.py does not exist
      in the project files. long_module.py calls
      self._pm.on_entry_filled() and self._pm.clear_record()
      — method names inconsistent with actual position_mirror.py
      committed to GitHub (which has create(), close_position()).
      signal_pipeline.py uses self._pm.all_records.
      risk_engine.py uses self._pm.all_records and open_count.
      Blocked from writing test_integration.py until
      position_mirror.py is uploaded to project.

Decision: Add pm_create/pm_on_fill/pm_on_orders/pm_clear
as WSManager instance methods. Remove position_mirror
injection from EE constructor. EE calls self._wm.pm_*().
WSManager remains SOLE WRITER. HR-PM-009 satisfied.

2.2 OI-001 Fix — ws_manager.py
------------------------------------------------------------

Changes made:
  (1) Added entry_limit_price: Decimal = Decimal("0") to
      PositionRecord dataclass after cl_ord_id field.
      Field needed by pm_create and by 1011006 spec.

  (2) Added 4 PM write methods to WSManager class:

        pm_create(symbol, cl_ord_id, entry_limit_price, qty)
          Creates PositionRecord at entry dispatch.
          HR-PM-002: record-at-dispatch, no async gap.

        pm_on_fill(symbol, fill_price, qty, timestamp_utc)
          Updates entry_fill_price, qty, timestamp on fill.
          HR-PM-003: uses actual avg_price, never limit price.

        pm_on_orders(symbol, tp_order_id, emgsl_order_id)
          Updates tp_order_id and emergsl_order_id on
          batch_add ACK.

        pm_clear(symbol)
          Deletes PositionRecord on entry cancel or expire
          with no fill (EE path only — not full position close).

  All 4 methods write to self.position_mirror directly.
  All 4 log their state change event.
  WSManager is SOLE WRITER to self.position_mirror.
