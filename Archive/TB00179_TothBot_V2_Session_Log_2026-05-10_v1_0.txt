OP-009 (rationale — historical recurrence pattern
that drove §4.2 standing rule).

Corrective taken: str_replace remediation +
post-remediation grep verification + this §7.1
log entry per RR-VIOL-002.

Recurrence count update: Cumulative OP-* recurrence
count incremented by 1. Forward tracking continues
through future NSI carry-forwards.

Root-cause analysis: The pre-emit scan in
authoring-time was insufficient because the
draft was being assembled in str_replace blocks
without a full-text grep until post-write
validation. The OP-003 specification says "Before
emitting any Windows file path in any response,
Claude scans the path for the substring..." —
the rule's literal text scopes to "Windows file
path" emission, but OP-002's general scope is
"all documents Claude produces (governance,
specifications, session logs, NSIs, DC packages,
revision histories, commit messages, delivery
notes)." Authoring revision history text within
a controlled document falls under OP-002 general
scope; the pre-emit scan needs to fire at the
str_replace draft level, not only at the
post-write validation level. Operational lesson:
when authoring controlled document content
containing references to §4.2 itself, grep the
draft before applying str_replace.
