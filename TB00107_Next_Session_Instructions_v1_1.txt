AI-1 — OI-023 DEFECT-TRADE-RECORD-001 (CRITICAL)
------------------------------------------------

Problem:
  TRADE_CLOSE records are written only to tothbot.log.
  tothbot.log is managed by RotatingFileHandler with
  limited retention (~6 days observed on VPS). Any trade
  closed more than ~6 days ago is permanently lost.
  Breaks CIATS 200-trade statistical floor, IRS Form
  8949 recordkeeping, and audit trail.

Fix specification (LOCKED by TB00107 Section 3.6 FP/DP
analysis — do not re-derive):

  Location:  /home/tothbot/records/trades_<YYYY>.jsonl
             where <YYYY> is the current UTC calendar
             year at time of write. Rolls over at
             2026-01-01 00:00:00 UTC -> trades_2027.jsonl.

  Contents:  TRADE_CLOSE records only. No other event
             types. Full record per 1011007 Section 7
             TRADE_CLOSE contract — identical payload
             as written to tothbot.log.

  Format:    JSONL (one JSON object per line, newline-
             delimited). Match existing tothbot.log
             serialization exactly. Decimal values as
             strings. Timestamps as ISO 8601 UTC.

  Retention: PERMANENT. No rotation. No truncation. No
             size cap. Append-only writes.

  Write contract:
    - On every TRADE_CLOSE event, write to BOTH:
      (a) tothbot.log via existing Logger path
          (unchanged)
      (b) trades_<YYYY>.jsonl via new direct-append path
    - Both writes MUST be non-blocking to the trading
      hot path (consistent with HR-LG-001).
    - If the trades_<YYYY>.jsonl write fails for any
      reason (disk full, permission error, etc.):
      log an ALERT-level event, send operator alert
      HIGH, but DO NOT block the Logger or the hot
      path. tothbot.log write proceeds normally. This
      preserves current behavior as a fallback.

  File creation:
    - Directory /home/tothbot/records/ created at
      startup if not present (os.makedirs exist_ok=True).
    - Annual file created on first TRADE_CLOSE of the
      calendar year with mode "a" (append; creates if
      absent).
    - File permissions: 0644 (rw for owner, r for group
      and others — same as tothbot.log).

  CSV export:
    - NOT in this session. Deferred to future dashboard
      session. Annual JSONL is the authoritative source;
      CSV is a transform.

Design freedoms for Claude Code to decide:
  - Whether the write path lives inside the existing
    QueueListener (writes to both sinks) or outside
    (direct append in the hot path with a lock) —
    recommend QueueListener pattern to preserve
    HR-LG-001.
  - Whether to add a new handler (e.g.,
    TradesRecordHandler subclass) or a branch in an
    existing handler.
  - Exact naming of the constant for the records
    directory (suggest TRADES_RECORD_DIR =
    "/home/tothbot/records").
