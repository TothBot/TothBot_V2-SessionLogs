be re-opened or re-asked. It splits into two halves:

3.1 HALF A — THE ALREADY-FILLED PORTION.
    Handled by the Exit module via ar:AR-054 (entry order
    partial fill + expiry/cancel, direction-symmetric
    protection). When a GTD post_only entry order
    partial-fills and the remainder cancels/expires
    (exec_type=expired OR canceled with cum_qty>0), a real
    unprotected stub exists with NO TP and NO emergSL;
    AR-054 dispatches market SELL (Long_Module) / market
    BUY reduce_only=true (Short_Module) to reduce the stub
    to zero.
    STATUS: ALREADY CANONICAL. AR-054 is already in D2's
    root governs list (live file line 2175) and defined at
    0500000 §4 lines 5169-5195. Companion ar:AR-053
    (per-module Pending Order Registry, capital-commitment
    tracking, §4 line 5122) is also already in D2 governs.
    NOTHING TO AUTHOR for Half A. Done.

3.2 HALF B — CANCEL THE UNFILLED REMAINDER ON SIGNAL
    DETERIORATION (the SIGNAL_DECAY entry-cancel trigger).
    Per DEC-065 Q2: "if the entry signal (SSS) deteriorates
    below threshold while the entry order is still working
    (unfilled / partially filled), cancel the unfilled /
    remaining qty; the Exit Controller owns any
    already-filled portion per the existing AR-054
    partial-fill handling."
    DEC-065 Q2 explicitly states (TB00442 SL §2.4 confirmed)
    that this trigger mechanism DOES NOT EXIST in the
    current entry criteria — it is NET-NEW DESIGN to be
    authored into 0511003 Execution_Engine_Specification +

    STATUS: RATIFIED-AS-A-DECISION, NOT YET AUTHORED into
    any diagram or spec. This is the open work.
