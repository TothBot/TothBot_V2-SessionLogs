1. METHOD (and an honesty note on method)
------------------------------------------------------------

The question "which element owns this token" is an ARCHITECTURAL-
FIT question, not a statistical one. There is no numeric sample
data attached to these tokens in 0500000. Inferential statistics
(Mann-Whitney, EWMA-on-samples, etc.) do NOT apply and were not
used; claiming them would be fabrication. The correct method is
First-Principles structural analysis plus grep evidence:

  (a) Count structured param: binding sites for each token
      (q1_do / consumes / produces / ciats_ownership fields).
  (b) Read the q1_do role of each candidate owning element and
      test whether the element's DESCRIBED FUNCTION consumes or
      computes that exact token.
  (c) Apply the settled exclusions: the D5/D6
      mod:CIATS_Parameter_Store universal producer is NOT an
      owning element (DEC-042); universal/shared machinery with
      no per-module owner stays in sec 9.17 (CF-TB00405-A).

GROUND TRUTH (live grep, dv1_123): all 17 tokens have ZERO
structured param: sites. "Unbound" means no structured binding
exists YET -- it does NOT mean no identifiable owner. Where a
single element's q1_do role demonstrably consumes the token, a
clean owner exists and a structure-adding migration (0911002
Option B) is warranted. Where the role evidence is absent or
split, NO clean owner exists and the token stays in sec 9.17.
