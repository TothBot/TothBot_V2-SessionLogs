AR-044 — STARTUP WARM-UP: ATR(14) AND EMA VIA REST
         GetOHLCData:
  On startup, before any pipeline execution, for each
  monitored pair:
  (a) GetOHLCData(pair, interval=5) → seed ATR(14).
      Use response[:-1]. Last 14+ committed candles.
      ATR seed = SMA of first 14 TRs.
      Subsequent: ATR_n = (ATR_{n-1}×13 + TR_n) / 14
  (b) GetOHLCData(pair, interval=60) → seed EMA(20)
      and EMA(50) for 1H HTF cache.
      Use response[:-1]. Last 100+ committed candles.
      EMA seed: first n values = SMA(n). Then alpha=2/(n+1)
      EMA formula: EMA_n = alpha×close + (1-alpha)×EMA_{n-1}
  (c) GetOHLCData(pair, interval=1440) → Regime Engine.
      Already specified. Also runs on startup (not just
      00:00 UTC) to establish current regime immediately.
  STAGGER: 1.1s between calls for the SAME pair.
  PARALLELISM: asyncio.gather() across DIFFERENT pairs.
  While ATR or EMA not seeded: pair is WARM_UP state.
  No entries for WARM_UP pairs. Gate 4 and Gate 8 block.
  WARM_UP exits when first candle-close processed post-
  seeding.
  Assign: 1011002 WM-STARTUP-001 through -008,
          1011014 SS-WARMUP-001 through -010.

AR-045 — OHLC CANDLE-CLOSE DETECTION MECHANISM:
  WS Manager maintains per symbol:
    last_interval_begin: dict[str, str]
    last_complete_candle: dict[str, OHLCCandle]
  On each ohlc(5m) message for symbol S:
    if msg.interval_begin == last_interval_begin[S]:
      → Update in-progress candle data. DO NOT fire pipeline.
    else:
      → Candle for last_interval_begin[S] is NOW CLOSED.
      → Fire pipeline with last_complete_candle[S].
      → last_complete_candle[S] = current message data.
      → last_interval_begin[S] = msg.interval_begin.
  Initialize last_complete_candle[S] from startup
  GetOHLCData(interval=5) response (most recent committed
  candle in response[:-1]).
  Same logic applies to ohlc(60) for HTF EMA cache updates.
  Assign: 1011002 WM-OHLC-001 through -006.

AR-046 — EMERGSL triggers.reference EXPLICIT:
  All emergSL stop-loss orders in batch_add MUST include:
    "triggers": { "reference": "last", "price": sl_price,
                  "price_type": "static" }
  Never rely on defaults for crash protection parameters.
  reference="last" means Kraken matching engine triggers
  when last traded price reaches sl_price.
  Deterministic behavior regardless of future API changes.
  Assign: 0411001 WS-SL-001 through -003,
          1011002 WM-BATCHADD-003 through -004.

AR-047 — PRICE AND QUANTITY ROUNDING BEFORE ORDER DISPATCH:
  Gate 8 MUST apply all four roundings before any order:
  (1) order_qty:
      order_qty = floor(raw_qty / qty_increment) × qty_increment
      If result < qty_min OR result × price < cost_min:
        → POSITION_TOO_SMALL. Skip pair. Log.
  (2) Entry limit_price:
      limit_price = floor(raw_price / price_increment) × price_increment
      Conservative: lower limit = more likely to fill
      post_only without taking.
  (3) TP limit_price:
      tp_price = ceil(raw_tp / price_increment) × price_increment
      Conservative: higher TP = better R:R at minimum.
  (4) emergSL trigger price:
      sl_price = floor(raw_sl / price_increment) × price_increment
      Result: slightly wider stop = more conservative
      emergency protection.
  All four must complete and validate BEFORE any add_order
  or batch_add is dispatched. If any rounding produces
  an invalid order (fails cost_min, qty_min, price
  crossing): log, alert, skip pair for this evaluation.
  NEVER dispatch a batch_add without pre-validated prices.
  Reason: batch_add validation failure kills entire batch
  → open position without TP or emergSL protection.
  Assign: 0411001 WS-ROUND-001 through -008,
          1011002 WM-ROUND-001 through -006.

AR-048 — EXIT CONTROLLER MAE USES BID PRICE FOR LONGS:
  For long positions, MAE detection and unrealized P/L
  calculation MUST use the `bid` field from ticker updates.
  NOT `last`.
  Rationale:
    - bid = best price at which the position can be
      immediately sold (realizable exit price)
    - last = most recent trade, can be stale
    - event_trigger:bbo was chosen precisely so bid
      updates fire immediately on adversarial price moves
  MAE formula: current_mae = entry_fill_price - bid_price
  Compare: if current_mae >= ATR(14)×1.5x → Layer 2 exit.
