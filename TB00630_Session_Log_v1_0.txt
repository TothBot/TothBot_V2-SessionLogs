============================================================
DECISIVE VERIFIED FACT (Kraken behavior, web-confirmed)
============================================================
A sub-minimum position CANNOT be sold on Kraken Spot. An order below the
minimum order size is rejected; an order below cost_min is canceled
("Cost minimum not met"). No reduce_only / market / closing-order exemption.
Sub-minimum balances are non-tradeable small balances; the only disposal
route is the manual "Convert Small Balances" UI feature (once/24h, 3% fee),
NOT automatable in the trading hot path.

CONSEQUENCE: WS-TP-003's "market-sell the remainder immediately" -- the
mechanic behind exit_reason TP_PARTIAL_FILL_REMAINDER -- is PHYSICALLY
IMPOSSIBLE (the disposal order is itself sub-minimum and is rejected). The
inherited architecture-of-record specifies an impossible action in the
loss-prevention domain. A force-close that silently fails is WORSE than no
force-close (phantom-state: Position Mirror cleared while the coin is still
held). This is a genuine latent DEFECT -> paired MDR entry required.

DECISIONS RATIFIED BY BILL (this session)
============================================================
DEC-A  SIZING. Wallet stays $5,000/side (paper). Trade-size seed changes
       from ~$0.50 to PAIR-RELATIVE:
         per_trade_size_usd = max(50, 5 * max(costmin, ordermin*price))
       read live per pair from the instrument channel. $50 floor so the
       entire liquid universe is tradeable; 5x margin above each pair's real
       floor so a partial fill cannot strand either side. Starting/seed only;
       CIATS-owned from the 200-trade floor. Rationale: capital recycles on
       spot, so trade size does NOT buy more trades (sample count is gated by
       signal frequency x time x concurrency, not size); size only controls
       fee/precision noise, dust formation, and R:R-floor validity. $0.50 was
       noise-dominated and dust-guaranteed; the data-clean knee is ~$30-50.

DEC-B  ENTRY EXECUTION MODEL. Replace the maker-only entry strategy
       (WS-ADD-005: post_only GTD 30-60s) with a TAKER entry: marketable IOC
       limit + Market Price Protection (MPP) slippage cap. TP exit STAYS
       maker (post_only resting limit). Protective exits (MAE/regime/emergSL)
       stay taker market (unchanged). Rationale (FP/DP): on a trend-filtered
       pullback signal, maker-only entry causes ADVERSE SELECTION (fills only
       when price ticks against you; misses the winners price runs to),
       yields a low/variable fill rate, and -- decisively -- produces a
       BIASED CIATS sample so the 200-trade certification measures the
       adversely-selected subset, not the strategy. Taker entry gives
       unbiased immediate fills, captures the moves, and ELIMINATES the
       entry-side partial-fill dust/unprotected-stub at the source. The fee
       delta is ~0.10-0.15%/trade (~$0.05-0.08 on $50) -- trivial vs the
       selection-bias cost. Does NOT violate the sacred constraint: 1:1.5 is
       the GROSS floor (hardcoded); net just lands slightly lower (~1.38-1.42x
       vs ~1.46x) and acceptance rule A1 still rejects below 1:1.5 gross.

DEC-C  FEE SEEDS. Correct FEE_TAKER_PCT / FEE_MAKER_PCT to the realistic
       starting (base, zero-volume) tier so the R:R-floor and Half-Kelly math
       are honest: conservative seed = alt base tier (taker 0.0040, maker
       0.0025); live-read per pair/tier refines. Round-trip basis becomes
       taker-in + maker-out (TP), with taker-in + taker-out for protective
       exits.

DEC-D  DUST REMEDIATION (corrects the original Option-A "general force-close"
       to the only physically-valid remedy). The sub-minimum residual cannot
       be sold -> ABANDON-AND-TRACK as a bounded write-off: cancel the
       (rejected-anyway) protective order, clear the residual from active
       position management, book the bounded loss (< cost_min, ~<$0.50) to
       CIATS, and surface accumulated dust for the operator's manual Convert
       Small Balances action. New exit_reason SUB_MIN_RESIDUAL_ABANDONED
       (general, condition-named) REPLACES TP_PARTIAL_FILL_REMAINDER
       (trigger-named, force-sell). Touches the DEC-065 7-value enum ->
       paired DEC + MDR (the invalid force-close defect).

DEC-E  DUST SCOPE NARROWED BY DEC-B. Taker IOC entry removes the 30-60s
       dangling post_only GTD that partial-fills-then-expires, so the
       entry-side dust case and AR-054's unprotected-stub case are largely
       obviated (IOC fills or kills the remainder immediately; a sub-min
       filled part is abandoned per DEC-D). The ONLY remaining normal-op dust
       source is a partial TP fill (maker) leaving a sub-min held remainder
       -- rare at $50/5x-floor sizing, bounded < cost_min. The heavy
       entry-redesign (formerly "gamma") is therefore unnecessary.

NOTE on SIGNAL_DECAY / AR-054: DEC-B removes the working post_only entry
order, so SIGNAL_DECAY (DEC-065 Q2, the pending-entry-order cancel trigger)
and AR-054 (GTD-expiry unprotected stub) must be RE-EVALUATED for the IOC
model in the D2 amendment -- this may touch DEC-065 Q2 and is flagged in the
change plan, not yet resolved.
