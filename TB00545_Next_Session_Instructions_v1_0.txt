1. PRIMARY MISSION (ONE atomic mission)
------------------------------------------------------------

Author TB00000 v2_79: a single governance amendment that
fixes the recurrent whole-session crash-and-lose-everything
failure. This is ONE primary mission, authorized as such by
(a) Bill explicit chat-turn directive locking this scope
(TB00544 session, "Yes, Lock TB00545 with this scope"), and
(b) RR-SCOPE-011 exception (b) governance-amendment-to-
session-management-rules treatment.

ROOT CAUSE (established in TB00544, not assumed):
Crashes present as the client error "Claude couldn't finish
this response. Try again in a moment," after which retry
produces nothing and the uploaded transcript is empty. That
signature = a SINGLE response turn dying mid-stream with no
committed output. The killer is that a whole session's work
(all documents + the entire close ritual) is generated inside
one oversized turn. DR-008.6 actively causes this: it mandates
the close (OCS + present_files for every document + SL + NSI +
GitHub box) in ONE turn - the most overloaded turn in the
session, hence the most likely to die. The protection is the
poison. The count-based one-mission cap was a falsified
hypothesis: sessions crashed at the same rate while obeying
it, because the cap never touched per-turn output size or
fixed cold-load cost.

2. SCOPE — the four enumerated changes (no others)
------------------------------------------------------------

2.1  REVISE DR-008.6 from one-mega-turn to INCREMENTAL
     PER-TURN DELIVERY. Each artifact is authored and
     delivered via present_files in its own small turn the
     moment it is complete. The "session close" becomes the
     last small delivery, not a mega-turn - by the time the
     session closes, every deliverable has already been
     committed across prior turns. There is no giant final
     turn left to die in. The FOUR-OPERATION ordering content
     (OCS / documents / SL+NSI / GitHub box) is preserved as
     an ORDERING of small committed steps, not a single-turn
     requirement.

2.2  ADOPT SMALL-TURN / DELIVER-AS-YOU-GO as the standing
     operating discipline. Keep every generation turn well
     under the output limit; stream large documents in
     sections across turns, delivering progressively. This
     elevates DR-011 v2_78 from "checkpoint" to the primary
     crash-PREVENTION mechanism (corrected from the TB00544
     mid-analysis position that called it damage-control
     only - with the single-turn-overload signature it is
     preventive).

2.3  REFORM THE READING CONTRACT (section 2) to a MINIMAL
     BOOT SET + load-on-demand. Mandatory session-open
     reading shrinks to TB00000 core + latest SL/NSI only.
     Namespace (0311001), workflow (0311002), and the full
     0500000 HTML are pulled ONLY when the mission touches
     them. This cuts the fixed cold-load cost that leaves no
     headroom and makes retry produce nothing. GitHub-is-
     public on-demand pull (DEC-097) already permits this.

2.4  ADD AN ANTI-REGROWTH META-RULE (H). When an incident
     occurs, FIRST ask whether an existing principle already
     covers it; if yes, log the incident under that principle
     - do NOT mint a new absolute standing rule. A new rule
     is authored only when no existing principle covers the
     failure. Without this brake, the document regrows to its
     pre-cut size and the crash loop returns.

DROPPED (do NOT enact): the self-monitored budget-stop rule
(TB00544 option C). It is unenforceable - the agent has no
reliable live readout of remaining context/tool budget, so a
stop keyed to that signal repeats the count-cap error class.
