7. DECISIONS / DERIVATIONS (transparent; none invents a value)
   (1) THE CANDLE MINIMUM. Image4 states "minimum 28 daily candles" - that is the ADX(14)
       sub-requirement. A COMPLETE classification also needs EMA50 (50 committed) and the ATR
       50-day percentile window, so the binding floor is max(28, htf_ema_long) = 50. The guard
       is computed from the actual seed (not hardcoded 28); GetOHLCData returns 720, always
       adequate. compute_regime raises RegimeComputeError (-> REGIME_COMPUTE_FAIL) below the floor.
   (2) THE DIRECTIONAL EMA TIE. Image4 writes the directional split with strict inequalities
       (POS EMA20>EMA50, NEG EMA20<EMA50) and defines NON_DIR solely by ADX<=threshold. An exact
       EMA20==EMA50 tie under ADX>threshold is a measure-zero Decimal coincidence the figure does
       not enumerate; classify resolves it to TRENDING_NEGATIVE - the loss-min read (blocks the
       LONG via the cascade rather than admitting a long into a non-rising HTF). Boundary
       completion, not a new threshold.
   (3) THE SHORT-SIDE SSS MIRROR - FLAGGED FOR BILL RATIFICATION. Image2 writes SC-SSS-2
       generically ("EMA9 > EMA21") and the short RSI bounds as the registry "mirror" of the
       long bounds, stating the engine is "direction-symmetric ... short-side per mod:Short_
       Module" without spelling the short inequalities literally. sss.py implements the only
       coherent symmetric reading: short SC-SSS-2 = EMA9 < EMA21, and the short RSI zone = the
       open interval between the two short bounds (min..max = 50..70). NO value invented (the
       bounds are the registry seeds rsi_short_low 70 / rsi_short_high 50); only the inequality
       direction is completed. If Bill wants the short side spelled literally INTO Image2, that
       is a one-line figure annotation (a Claude Code diagram edit) before any short-module wiring.
