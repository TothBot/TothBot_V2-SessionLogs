============================================================

  - .produces field carries DATA-FLOW TOKENS (the data
    emitted by the producer node).
  - q1_do field carries the producer's behavioral
    description.
  - q4_triggers field carries the emission trigger
    semantics (when/why each token emits).
  - q5_logs field carries the persistence/alert semantics
    (where each token is logged + at what level).

canonical token references with trailing prose qualifiers
("on every Kelly recompute", "CRITICAL on K_full <= 0 per
HR-CI-008") — a V-040.2 Q2 schema-type regression where
behavioral narrative was embedded in the data-token
field. Coders reading .produces would either parse the
trailing prose as additional data tokens (incorrect) or
treat them as condition-rules embedded in the data field
(also incorrect — emission triggers belong at q4_triggers,
logging semantics at q5_logs per §15.16.3 cell taxonomy).

Post-amendment, .produces contains only schema-type-correct
CSV tokens. Behavioral semantics live at the
