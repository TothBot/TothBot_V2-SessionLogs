4.1 SCOPE DECISION GATE

Bill mid-session question: "Do we want to continue to
use .txt files for most and HTML for some (i.e. 0500000)?
Or should we switch to HTML for all documents?"

Claude FP/DP analysis returned (response in chat):

  RECOMMENDATION: Keep the v2_20 split. HTML for
  diagram-bearing docs only. .txt for everything else.

  Six hidden problems HTML-for-all would introduce:
    (1) HTML escaping defects (every < > & in prose
        becomes a new edit-time defect risk).
    (2) Tag-balance regression on every str_replace
        edit (.txt has no structure to break).
    (3) GitHub diff noise (HTML diffs interleave tag
        boilerplate with content changes).
    (4) Code spec / .py header mismatch impedance.
    (5) Session log / NSI authoring cost increase.
    (6) ~80-doc migration cost for zero capability
        gain on prose docs.

  Why split is principled: pure-prose .txt docs have
  zero defect history in this project. The HTML
  advantage is the Claude.ai preprocessor's inline-SVG
  visual rendering — applies only to diagram-bearing
  docs.

Bill APPROVED: "I approve the Recommendation: keep the
v2_20 split. HTML for diagram-bearing docs only. .txt
for everything else."
