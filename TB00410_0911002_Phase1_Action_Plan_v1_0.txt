2. PER-SESSION FP/DP RE-VERIFICATION GATE (MANDATORY, ALL
   SESSIONS A/B/C/D) -- BILL DIRECTIVE TB00411-CHAT
------------------------------------------------------------

Each session (A, B, C, D) MUST, at session open and BEFORE any
edit, run an INDEPENDENT FP/DP analysis SCOPED TO THAT SESSION'S
SPECIFIC ITEM ONLY, re-derived from the live 0500000 -- not from
this plan and not from the TB00410 Determination. This extends
and operates together with the DD-011 CHAT-TURN-1 GATE.

The TB00410 Determination disposition is the PRIOR INPUT to this
gate, never its authority. The session-open re-derivation is the
authority. No session executes on memory or on the prior
determination alone.

GATE STEPS (per session, item-scoped):

  RV-1  RE-GREP. Grep the live 0500000 for that session's
        specific token(s) only. Confirm the current bound /
        unbound state and the exact structured param: site
        count. Confirm bare-name occurrence contexts (prose /
        sec 9 mirror / SVG label / revision history).

  RV-2  RE-READ THE ROLE. Read the candidate owning element's
        live q1_do / consumes / produces / governs fields.
        Confirm the element's DESCRIBED FUNCTION consumes or
        computes that exact item.

  RV-3  RE-RUN THE FP/DP. Independently re-derive the
        owning-element analysis for that one item: enumerate the
        option set (CHOICE 1 single owner / CHOICE 2 new diagram
        / CHOICE 3 stay sec 9.17 / UNDETERMINED), apply
        first-principles fit, eliminate, and apply the
        single-owning-element test. Honor the DEC-042 universal-
        producer exclusion.

  RV-4  COMPARE TO PRIOR DISPOSITION.
        - MATCH  -> proceed with the migration as planned.
        - DIVERGE -> HALT. Do NOT edit. Surface the divergence,
          record the corrected disposition, and treat the
          session as a re-determination (no migration) unless
          the corrected disposition is itself a clean derivable
          migration with Bill direction intact.

  RV-5  RECORD. The session log records the RV-1..RV-4 result
        for the item, with the live grep evidence, so the
        migration (if it proceeds) rests on a current,
        independent re-derivation rather than on this plan.

This gate is a NECESSARY precondition for every migration in
Sessions A-D. A session that skips it, or that edits before
RV-4 returns MATCH (or a Bill-directed corrected disposition),
is a session-halting defect under RR-VIOL-001.
