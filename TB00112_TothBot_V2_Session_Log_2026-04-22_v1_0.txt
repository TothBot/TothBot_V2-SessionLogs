------------------------------------------------------------
6.1 Pre-flight version sweep MUST be automated
------------------------------------------------------------

Manual head-of-file version reads caught the logger.py
and startup_sequence.py drift this session, but only
after Claude had already written instructions based
on stale data in TB00111 NSI. A simple bash one-liner
on VPS and a parallel sweep on the project folder
would catch all drift in 5 seconds.

Lesson: every future NSI §5 pre-flight MUST run a
complete DocVersion sweep of both sides and compare
all files, not just the two or three mentioned by
name. Stated from first principles: memory-stated
version numbers are not fresh data.

------------------------------------------------------------
6.2 Governance paths are authoritative - operational
    notes in NSIs are not
------------------------------------------------------------

The OneDrive Desktop path documented in TB00111 NSI §9
is an operational note, not a governance rule. When
it conflicts with 03XXXXX procedure (as it did this
session), the 03XXXXX procedure wins every time.

Lesson: before issuing a command that uses any local
path, Claude MUST verify that path is in the 03XXXXX
series. If it is not, Claude MUST use a governed path
or flag the gap to Bill.

------------------------------------------------------------
6.3 GitHub is the authoritative source for code - NOT
    the VPS
------------------------------------------------------------

Even though the VPS has the same byte-identical files
as HEAD on GitHub (by virtue of being deployed from
HEAD), the VPS is downstream of GitHub. Pulling a
file from VPS and treating it as authoritative
short-circuits GitHub as the source of truth, which
is a direct violation of TB00000 §11. The correct
pattern is always:

  GitHub -> local repo (git pull) -> Claude.ai project

Never:

  VPS -> Desktop -> Claude.ai project

------------------------------------------------------------

Two Claude operational errors this session (§2.6).
Both are documented with the specific wrong
instruction, Bill's correction, and the recovery
path. Future-session Claude reads this log and sees
