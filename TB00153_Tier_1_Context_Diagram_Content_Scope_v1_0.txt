4.3 PRIMARY DATA / EVENT FLOWS — DEPICTED ARROWS
------------------------------------------------------------

Each arrow becomes an SVG <g class="arrow"> with
<title>+<desc>. Six primary flows MUST be depicted:

  Flow 1  Kraken WS v2 public push -> TOTHBOT
          (5-min OHLC candle close = system clock).
          Bidirectional possible at subscribe-arm.

  Flow 2  Kraken WS v2 private push <-> TOTHBOT
          (live mode only; paper mode SUPPRESSED
          per PA-004(1)).

  Flow 3  Kraken REST <-> TOTHBOT (instrument
          snapshots; warm-up OHLC; ticker batches).

  Flow 4  TOTHBOT <-> CIATS (Trade Outcome Bus
          publishes; CIATS Parameter Store reads
          consumed by TOTHBOT modules).

  Flow 5  TOTHBOT -> alerts@tothbot.com (alert
          events out of Logger).

  Flow 6  Bill <-> Hetzner VPS (operational SSH);
          Bill <-> GitHub (commits + reads).

Direction-of-mimicry annotation MUST appear visually:
  Paper IS MASTER. Live MIMICS Paper. The PA-004
  divergence point arrows on Flow 2 + Flow 3 MUST
  be visually marked as the ONLY differences between
  paper and live operation.

============================================================
6. T-2+ TAGGING CONTRACT
============================================================

Per 0311001 v1_6 §15.11.3:

  - Every labeled <g> group MUST carry <title>
    immediately followed by <desc> with structured
    key:value body.
  - <title> follows the canonical:short_name pattern
    (container:..., actor:..., arrow:...).
  - <desc> body MUST include canonical:, role:,
    and (where applicable) drill_target:, rule:,
    constraint:, boundary:, schema_version:.
  - Decorative groups marked class="decoration"
    are exempt from <title>+<desc> requirement
    but recommended for accessibility.
  - The diagram-level <title> at <svg> root MUST
    be the canonical name "0500000_Context_T1_R0_
    System_Context".
