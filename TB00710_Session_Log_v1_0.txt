     silent_pair.py - the per-pair silent-pair state machine (WM-SHARD-010 +
       WM-PS-007 + HR-WM-030). A pure timer over an injected monotonic clock.
       T_SILENT_SEC = 60.0 (fixed engineering constant). States INITIAL /
       SUBSCRIBED / DATA_READY / DATA_PENDING; the five drawn transitions exactly:
       INITIAL -[sub ack]-> SUBSCRIBED; SUBSCRIBED -[update <= T_silent]->
       DATA_READY (no event); SUBSCRIBED -[no data > T_silent]-> DATA_PENDING
       (emits evt:PAIR_DATA_PENDING at evaluate()); DATA_PENDING -[update]->
       DATA_READY (emits evt:PAIR_DATA_READY_RECOVERED); (any) -[shard reconnect]->
       SUBSCRIBED (re-arms the timer). gate_blocks() returns True ONLY in
       DATA_PENDING (HR-WM-030 Pre-Gate-1 skip); the machine MARKS but never
       removes a pair (HR-WM-031 anti-AR-070 held-not-evicted - no eviction API).
       Boundary discipline: update <= T_silent is READY, evaluate at exactly
       T_silent does NOT fire, a data frame racing in just after expiry still
       resolves to DATA_READY (data presence wins - loss-min). Marker/poll methods
       return the event to emit (or None), mirroring keepalive.liveness().
