PURPOSE
  The exhaustive fact-level gap register for the SECOND D1
  source, 0411002 Kraken_REST_API_Specification dv1_1, against
  the live D1 figure (Kraken API Connection Architecture) in
  0500000 dv1_151. It enumerates every architectural wire-fact
  carried by 0411002 and marks each PRESENT-IN-D1, ABSENT, or
  RECONCILE, where "in D1" means derivable from the D1 figure
  ALONE per the strict DD-014 standard (SVG visual <text> +
  governs <desc> descriptors + figcaption ONLY; NO 0500000
  prose, NO section-8 AR registry, NO subordinate spec).

  This is the REST counterpart to the TB00530 D1 gap register
  (which covered the FIRST D1 source, 0411001 WS v2). The 37
  TB00530 0411001 entries are now all resolved and D1 is
  fact-level comprehensive FOR 0411001 (dv1_143..dv1_150,
  closed at TB00539; 0411001 retired TB00541). 0411002 is a
  DISTINCT source: the REST wire contract was never migrated,
  so most REST-specific facts are absent from the diagram-alone
  corpus even though several REST-ADJACENT facts already landed
  via the 0411001 migration (HMAC signing model WS-AUTH-002,
  GetOHLCData [-1] exclusion HR-WM-013, the 8-channel
  drill_targets enumeration, spot-only balance WS-BAL-002).

  This register is read-only and is the input that drives the
  one-amendment-per-session D1 REST migration (CLUSTERS R1..R6
  below) and the eventual DR-078-gated 0411002 retirement.
  Read-only this session: no 0500000 edit, no retirement, no
  git rm, no C: operation (sec 4.4 CD-001).

  both repos PUBLIC. The D1 "diagram-alone" corpus = the

  b5e3d488c940759f4e390ba13719367f, byte-confirmed at cold-load

  BINARY TEST (DEC-096): every 0411002 element is TothBot-

============================================================

  R-D8  GetOHLCData error/retry. 0500000 section 2 PROSE (A-24,
        dv1_10) states the RATE-LIMIT (HTTP 429) path: exponential
        backoff, "retry up to 3 times", then use stale regime.
        0411002 REST-OHLC-009 states the GENERAL-error path:
        "retry once after 2 seconds", then use prior regime.

        general), not a contradiction - but the retry COUNT differs
        (3 vs 1) and neither REST retry policy is in the diagram-
        alone corpus. At CLUSTER R2 migration: confirm the two
        paths are distinct against the canonical 0411002 source +
        Kraken docs (SD-3, diagram is authority); do NOT silently
        copy either value. NO-GUESS: not asserted as a CONFLICT
        without same-path confirmation.
