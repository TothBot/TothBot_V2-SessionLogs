2.2 Defect Diagnosis — Two Critical Defects Confirmed
------------------------------------------------------------

DEFECT-WM-CRASH-001 (CRITICAL — CONFIRMED)
  Description: Recurring TaskGroup fatal crash. Service
    crashes and systemd restarts every few hours.
  Evidence: WS_MGR_FATAL + WSMGR_FATAL_PROPAGATED logged
    at 17:06:22 UTC today. Pattern across full log:
    at least 4 crashes on April 13-14 (10:27, 11:12,
    17:06 UTC and prior runs).
  Root cause (partial): The except block in run() logs
    only str(exc) — for a Python 3.11+ ExceptionGroup
    (raised by asyncio.TaskGroup on task failure) this
    produces only "unhandled errors in a TaskGroup
    (1 sub-exception)". The actual sub-exception is
    permanently lost. Cannot diagnose the real crash
    cause without this fix.
  Status: Fix 1 applied this session. Next crash will
    log the actual sub-exception and traceback.

DEFECT-RE-002 (CRITICAL — CONFIRMED)
  Description: Regime engine never seeded at startup.
    regime_cache empty for all 50 pairs. Gate 3 returns
    NO_REGIME_DATA and blocks all entries.
  Evidence: Zero regime computation events in the entire
    log. All pairs failing Gate 3 since first candle
    close after startup. System has processed multiple
    candle cycles but zero trades possible.
  Root cause: _trigger_daily_regime_refresh() is only
    called at 00:00 UTC (ohlc(60) handler). It is never
    called at startup. regime_cache is empty from startup
    until 00:00 UTC — potentially 23+ hours with no trades.
  Status: Fix 2 applied this session.

DEFECT-WM-LIQ-001 (WARN — LOGGED, NOT FIXED THIS SESSION)
  Description: Liquidity refresh returning empty_result
    every 4 hours. Prior cache values preserved — Gate 2
    functional. Non-blocking.
  Evidence: LIQUIDITY_REFRESH_FAILED at 05:26, 09:26,
    15:12 UTC today.
  Status: Deferred. Will investigate after crash root
    cause confirmed from Fix 1 log output.
