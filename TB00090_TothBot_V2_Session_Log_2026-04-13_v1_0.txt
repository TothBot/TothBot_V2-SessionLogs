2.1 DEFECT-SS-002 — Missing position_mirror in
    ExecutionEngine() call
------------------------------------------------------------

Symptom:
  [STARTUP FATAL] ExecutionEngine.__init__() missing 1
  required positional argument: 'position_mirror'
  Crash loop immediately on service start.

Root cause:
  startup_sequence.py _build_components() called
  ExecutionEngine(ws_manager=wm, risk_engine=re, logger=logger)
  without position_mirror=pm. ExecutionEngine.__init__()
  requires position_mirror as a mandatory positional argument.

Fix:
  Added position_mirror=pm to ExecutionEngine() call.

File:    tothbot/startup_sequence.py
Version: dv1_3 → dv1_4
Commit:  b8fb0a8 (TothBot_V2-Code)

------------------------------------------------------------
2.2 DEFECT-SS-003 — PAPER_TRADING_MODE env var never
    read by _load_config()
------------------------------------------------------------

Symptom:
  After DEFECT-SS-002 fix, service started but immediately
  crashed with WS_MGR_FATAL: GetWebSocketsToken error:
  ['EAPI:Invalid key'].
  COMPONENTS_WIRED confirmed, paper mode gate in
  ws_manager.py was not activating.

Root cause:
  _load_config() in startup_sequence.py never read
  PAPER_TRADING_MODE from os.environ. The key
  "paper_trading_mode" was absent from the config dict
  returned to WSManager. WSManager.paper_mode was always
  False regardless of the env var. Private WS auth
  (_rest_get_ws_token) was always called. EAPI:Invalid key
  crash because paper trading API keys have no trade
  permissions.

Fix:
  Added to _load_config() return dict:
    "paper_trading_mode": os.environ.get(
        "PAPER_TRADING_MODE", "false").lower() == "true"
  Added PAPER_TRADING_MODE_ON / PAPER_TRADING_MODE_OFF
  log event in _async_main() after config load.

File:    tothbot/startup_sequence.py
Version: dv1_4 → dv1_5
Commit:  8cbc6f1 (TothBot_V2-Code)
