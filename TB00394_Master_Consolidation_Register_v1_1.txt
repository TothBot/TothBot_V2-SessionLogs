§PHASE-MODEL — UNIFIED OCS 4-PHASE ROADMAP (v1_1)
====================================================================
The campaign uses the unified OCS 4-phase model. This
SUPERSEDES the prior 0911002 5-phase numbering inside this
register. Formal enactment of the unified model in plan
0911002 is a separate carry-forward session per RR-SCOPE-011.

  Phase 0  SCOPE DEFINITION. This register. Document-level
           MOVE/STAY/SPLIT classification against the scope
           boundary. Bill-ratified at v1_1.
  Phase 1  ELEMENT AUDIT. Per-document element-level audit
           (DD-012 element identification). One source
           document per session per RR-SCOPE-011. Not started.
  Phase 2  DD-012 THREE-BUCKET + SEQUENCING. Per element:
           (a) architectural restatement -> drop with
           retirement; (b) coding-unique -> migrate to parent
           AA spec or preserved AA=10 spec; (c) architectural
           content not yet in diagrams -> amend 0500000
           diagrams first. Execution order determined here.
  Phase 3  EXECUTION. Diagram amendment, prose-follows-diagram
           reconciliation, instance-spec migration, AND
           anchor/domain-doc collapse + governance
           reconciliation. The old 0911002 Phase 4
           (anchor/domain collapse) is folded into Phase 3.

  OLD-NUMBERING -> UNIFIED-MODEL MAP (for the inline
  per-document tags in this register):
    old 0911002 Phase 0 (D8-closure) ........ closed history
    old Phase 2 (AA=05 component specs) ...... Phase 3 / E1
    old Phase 3 (AA=04/06/07/08 instance) .... Phase 3 / E2
    old Phase 4 (anchor/domain collapse) ..... Phase 3 / E3
  E1/E2/E3 are Phase 3 execution sub-stages preserving the
  dependency order (component specs migrate before instance
  specs; instance specs migrate before anchor/domain collapse).

  MOVE   Architectural; consolidate into 0500000. Target
         diagram / section / §9 mirror specified.
  STAY   Governance (AA=03), pure code (AA=10), or

         controlled document.
  SPLIT  Mixed. Architectural part MOVEs; structural-anchor
         or process remainder STAYs / is decided in Phase 3
         execution sub-stage E3 (anchor/domain collapse).

         ratified per TB00468 Decision 1. No document remains
         FLAG (count 0).
