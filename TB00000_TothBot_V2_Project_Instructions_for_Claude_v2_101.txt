1. IDENTITY AND MISSION
============================================================

TothBot V2 is an automated cryptocurrency spot trading system
on Kraken, written in Python. Bill is the sole operator. It is
a clean-slate rebuild; nothing from the old Binance-era TothBot
carries over. Old repos (TothBot-Docs, TothBot-Code) are
read-only archives, reference only. When in doubt, derive from
first principles.

It is a three-system organism operating at three time scales:
  Kraken  - event source
  TothBot - event processor
  CIATS   - continuous-improvement engine
CIATS owns every operating parameter. The only hardcoded
constraint is the net 1:1.5 R:R minimum.

MISSION: maximize profit, minimize loss - with emphasis on
minimizing loss. Loss prevention takes priority over profit.
Everything else is secondary.

7. TRADING SYSTEM
============================================================

Exchange: Kraken Spot. Language: Python 3.12.3. Clock: Kraken
5-minute OHLC candle close (WS v2 push).

PARALLEL MODULES. TothBot trades through independent sibling
modules - Long_Module, Short_Module, and any future modules.
Each has its own universe, gate thresholds, signal parameters,
trading wallet, sizing function, CIATS instance + statistical
pool, and per-wallet drawdown halts. Modules share only the
single Kraken WebSocket connection at the data layer (one WS
manager subscribes to the union of module universes - required
by Kraken rate limits). Everything above the WS manager is
module-independent. No cross-module data contamination.

WALLETS. Each module's wallet is operator-funded with what Bill
can afford to lose on that side - this is the primary loss
control. CIATS observes the balance and sizes against it; it
does not own the balance. Paper starting balances are synthetic
($5,000 long / $5,000 short); live funding is a separate Bill
decision.

EXITS (Exit Controller owns all normal exits):
  Layer 1a - regime-reversal exit: THE take-profit (run to
             reversal; no fixed TP price, no resting TP
             order, no max-hold time cap). DEC-124.
  Layer 2 - MAE threshold breach (1.5x ATR).
  Layer 3 - Kraken resting stop-loss: emergency brake only;
            fires ONLY on VPS/TothBot/internet failure; must
            never fire in normal operation.

SACRED CONSTRAINT - net (after all fees) 1:1.5 R:R minimum. The
minimum expected
reward-to-risk on every trade and the only hardcoded value. It
is an entry-acceptance floor ONLY - never a cap, never an exit
price (DEC-124).
CIATS may never propose, size, or accept below it. CIATS owns
the sizing curve ABOVE the floor (higher expected R:R may get
larger allocation). The module wallet balance is the sole
sizing boundary - no ceiling, cap, or limit on trade size.
Acceptance rule A1: reject any candidate whose point-estimate
expected R:R is below 1:1.5.
FLOOR GOVERNANCE: the floor is an operator-controlled hardcoded
MINIMUM - NEVER lowered below 1:1.5, but RAISABLE by Bill (e.g. ->
1:2.0) on CIATS evidence that a higher net (after-fee) floor
preserves sufficient trade flow and improves net edge. CIATS
monitors/analyzes the floor-vs-trade-count-vs-net-edge trade-off
and surfaces a floor-raise recommendation; Bill ratifies (a
hardcoded-constant change, never a CIATS runtime tune).

All other parameters are CIATS-owned starting values,
instantiated per module, replaced by data over paper trading.
Notifications go to alerts@tothbot.com.
