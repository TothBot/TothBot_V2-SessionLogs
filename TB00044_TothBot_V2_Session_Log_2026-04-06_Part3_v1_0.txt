Key design decisions:

  atr_14 recovery:
    Signal pipeline gate_8 output includes mae_pct but
    not atr_14 directly. atr_14 recovered at dispatch:
      atr_14 = mae_pct * entry_limit_price / mae_mult
    This is mathematically exact (inverts the gate_8
    mae_pct formula). Stored in _entry_orders context.
    Used for post-fill gate_8 recomputation (EE-BA-002).

  Post-fill gate_8 call:
    On entry fill, EE calls risk_engine.gate_8() with
    actual avg_price and stored atr_14. Uses tp_price
    and emergsl_price from result. entry_qty from fill
    event (cum_qty) — not recomputed. sizing_modifier
    passed in extended_spec so PL-NEW-005 fix applies.

  cl_ord_id format:
    Entry:  symbol[:6] (no slash) + timestamp_ms (EE-ID-001)
    TP:     "T" + 10-digit counter + symbol[:5]
    emergSL: "S" + 10-digit counter + symbol[:5]
    All guaranteed unique via monotonic counter.

  Partial fill on GTD expiry (EE-REJ-002):
    cum_qty > 0 → synthesises filled-style event and
    delegates to _on_entry_filled(). TP+emergSL placed
    for the partial qty.

  batch_add response:
    Handled by on_batch_add_response(msg). Extracts
    Kraken-assigned tp_order_id and emgsl_order_id.
    Calls pm.on_batch_add_ack(symbol, tp_id, emgsl_id).
    Cleans up req_id_registry.

  Rate limit rejection:
    on_rate_limit_reject() logs ENTRY_REJECTED_RATE_LIMIT.
    Caller (WS Manager) handles PM/semaphore cleanup.
