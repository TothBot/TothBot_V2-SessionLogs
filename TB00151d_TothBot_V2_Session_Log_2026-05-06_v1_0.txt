2. ROOT CAUSE OF FORMAT PROLIFERATION
============================================================

The 0500000 source-of-truth set had grown to 13 files for
ONE controlled document:
  1  .txt Claude Source
  1  .docx
  1  .pdf
  4  _svg.txt SVG sibling files
  6  .png raster sibling files

Each new format was added to patch a problem the prior
format created:

  (1) .txt picked as canonical because plain-text-editable.
      -> .txt cannot show diagrams when opened.
  (2) .docx added so Bill could view diagrams embedded.
      -> .docx settings.xml validation gate failures
         recurring across sessions (TB00130 origin).
  (3) .pdf added as backup viewable format.
      -> Three formats now diverging across regenerations
         (ERROR-TB00149-02 SVG-DROP root cause).
  (4) _svg.txt added to store SVG vector sources separately
      (CI-002, CI-003 prohibit inline binary embeds).
      -> Bill must rename .svg <-> _svg.txt cyclically to
         view locally vs upload. Multiple delete-and-re-
         upload cycles resulted (TB00151b/c).
  (5) .png added to store raster versions for .docx embed.
      -> 13 files, three viewers, two rename cycles, no
         single source of truth Bill can open and read.

Fundamental mismatch: the canonical format (.txt) does not
natively display embedded content. Every additional format
was a patch to compensate. Compound patching produced the
13-file proliferation.

FIRST PRINCIPLES RESOLUTION:

Pick a canonical format that satisfies ALL of:
  (a) on the Anthropic-supported project upload list,
  (b) plain-text editable and git-diffable,
  (c) renders embedded SVG natively when Bill double-clicks,
  (d) extracts cleanly as text for Claude to read,
  (e) does not require sidecar asset files.

Web search and Anthropic official help-page fetch
confirmed the supported project upload formats:
  Documents: PDF, DOCX, CSV, TXT, HTML, ODT, RTF, EPUB,
             JSON, XLSX
  Images:    JPEG, PNG, GIF, WebP
Source: https://support.claude.com/en/articles/
        8241126-upload-files-to-claude

.svg is NOT on either list — confirms the empirical
rejection observed in TB00151b/c.
.html IS on the document list — qualifies.

HTML is the ONLY format on the supported list that
satisfies (a) through (e) simultaneously. Browsers render
inline <svg> as a native HTML5 standard. The same file
opens for Bill (browser renders text + diagrams), uploads
to project knowledge unchanged, and reads cleanly as text
for Claude.
